# CTA Copy Patterns

## Role in LP

CTAボタンの文言で、行動率を最大化する。

## Required Elements

- 行動動詞
- ベネフィット
- 心理的ハードル低下

## High Conversion Pattern

### 「動詞 + ベネフィット」型
- 「無料で資料を受け取る」
- 「3分で診断する」
- 「今すぐ予約する」
- 「14日間無料で試す」

### 「ハードル低減」型
- 「クレカ不要で無料体験」
- 「1分で完了 / 資料DL」
- 「メールアドレスだけで受け取る」

### 「具体的成果」型
- 「無料で試算する」
- 「自分の市場価値を診断する」

### 「カジュアル」型（採用 / コンサル）
- 「カジュアル面談を予約する」
- 「相談だけしてみる」

## Bad Pattern

- 「お問い合わせ」（抽象 / ハードル高）
- 「送信」（行動が分からない）
- 「クリック」（不適切）
- 「無料相談はこちら」（受動的）
- 「Submit」（英語のみ）

## Copy Examples

### BtoB SaaS
- 「無料で資料を受け取る」
- 「14日間無料で試す（クレカ不要）」
- 「30分のデモを予約する」

### 採用LP
- 「カジュアル面談を予約する」
- 「メンバーと話してみる」
- 「3分でエントリーする」

### EC / 物販
- 「今すぐ購入する」
- 「カートに入れる」

### コンサル / 高単価
- 「無料相談を予約する」
- 「個別相談（無料）に申し込む」
- 「資料を請求する（無料）」

## Design Notes

- 行動の前にベネフィットを置く
- 「無料」「クレカ不要」を強調（ハードル低減）
- 文字数は10〜15文字程度（読みやすさ）

## Measurement Notes

- ボタン文言を `data-cta-text` で記録（A/Bテスト用）

## Checklist

- [ ] 行動動詞含む
- [ ] ベネフィット明確
- [ ] 「お問い合わせ」だけになっていない
- [ ] 文字数適切（10〜15）
