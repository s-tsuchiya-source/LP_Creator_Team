# Future State Section Patterns

## Role in LP

導入後の未来を描き、ユーザーが「導入後の自分」を具体的に想像できるようにする。

## Best Use Cases

- 解決策セクション後半
- 「導入後の流れ」セクション
- 「あなたの業務がこう変わる」セクション
- 採用LPの「ここで働く未来」

## Required Elements

- 4つの未来（現状不満 / 放置 / 導入後 / 成果後）
- 時系列（1ヶ月 / 3ヶ月 / 6ヶ月）
- 一人称表現
- 数値根拠
- 「個人差」明記

## High Conversion Pattern

### 型A: 時系列型（BtoB SaaS）

```markdown
## 導入後、こう変わります

### 導入1ヶ月後
業務に慣れ、初期効果を実感
- 業務工数: 約20%削減

### 導入3ヶ月後
本格的な効果が見え始める
- 業務工数: 約60%削減（自社調べ・100社平均）
- 抜け漏れ: ほぼゼロ

### 導入6ヶ月後
組織全体での効果実感
- 経営層への報告自動化
- 部門間連携の改善

※ 効果には個人差・業種差があります
```

### 型B: 1日の流れ型（採用LP）

```markdown
## ここで働く、あなたの1日

### 9:30
朝会で当日のタスクを確認
リモートでもオフィスでも自由

### 10:00〜12:00
集中して開発
モダンな技術スタック（Next.js / Go / GCP）

### 12:00
ランチはメンバーと（オフィスの場合）

### 13:00〜15:00
ペアプロやコードレビュー
「分からない」を気軽に聞ける文化

### 15:00
1on1（マネージャーと）

### 17:00
進捗共有

### 18:00
退社。プライベート時間の確保
```

### 型C: Before / After + 周囲の反応型

```markdown
## 業務が変わると、周囲の反応も変わります

### 導入前
- 手作業で疲弊
- 上司「報告まだ？」と急かされる
- 経営層「データが古い」

### 導入3ヶ月後
- 自動化で時間に余裕
- 上司「楽になったね」と評価
- 経営層「リアルタイムで見える」

### 導入1年後
- 業務改善の主導者として認知
- 採用部門のキーパーソン
- 次のポジションへステップアップ
```

### 型D: 心理変化型（BtoC / 教育）

```markdown
## あなたの気持ちが、こう変わります

### 受講前
「自分にできるか不安」
「失敗したくない」
「本当に転職できる？」

### 受講中
「学びが楽しい」
「一歩ずつ進んでいる実感」
「メンターのおかげで挫折しない」

### 受講後
「ポートフォリオができた」
「面接で堂々と話せる」
「キャリア選択肢が広がった」

※ 効果には個人差があります
```

## Bad Pattern

### NG型: 抽象的「効率化」だけ

```
## 導入後

業務が効率化されます。
時間が節約できます。
チームが活性化します。
```

→ 想像できない。具体性なし。

### NG型: 三人称（「ユーザーは〜」）

```
## 導入後

ユーザーは効率化を実感します。
業務が改善されます。
```

→ 自分ごと化できない。

## Copy Examples

### 良い例
- 「朝、ダッシュボードを開くだけで状況が把握できる」
- 「上司から『分かりやすい』と評価される」
- 「残業せずに帰れる金曜日」
- 「半年後、組織のデータ活用文化が定着」

### 悪い例
- 「効率化」「最適化」（抽象）
- 「劇的に変わる」（誇大）
- 「絶対に成功」（保証）

## Design Notes

- 時系列は左→右（PC）または上→下（SP）
- アイコンで視覚化
- Before / After は対比的に
- 数値はバッジ風

## Measurement Notes

- セクション閲覧時間
- 各時期のスクロール深度

## Checklist

- [ ] 時系列（1ヶ月 / 3ヶ月 / 6ヶ月 等）
- [ ] 一人称
- [ ] 五感に訴える表現
- [ ] 周囲の反応
- [ ] 数値根拠
- [ ] 「個人差」明記
