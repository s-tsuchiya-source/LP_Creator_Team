# Form Field Reduction

## Role in LP

フォーム項目数とCV率は逆相関。必須項目を最小化する戦術集。

## Field Count vs CVR

| 必須項目数 | 期待CVR傾向 | 用途 |
|---|---|---|
| 1〜3 | 高 | EC初期入力 / 簡易資料DL |
| 4〜6 | 中〜高 | 標準BtoB資料請求 |
| 7〜9 | 中 | BtoB商談リード（質重視） |
| 10+ | 低 | リード質が極めて重要な場合のみ |

## Reduction Strategies

### 1. 任意化
- 「ご相談内容」「興味分野」は任意
- 「電話番号」をメール代替で省略可能に

### 2. デフォルト値
- 「お問合せ目的」をプルダウン+デフォルト選択
- 「ご相談内容」のテンプレ文

### 3. 自動補完
- 郵便番号 → 住所自動入力
- 会社名 → 業種推定（API利用）

### 4. マルチステップ
- 1ステップ目: 名前+メール（ハードル低）
- 2ステップ目: 詳細情報（決断後の入力）

### 5. 段階的取得
- 1次フォーム: 最小限（メール + 業種）
- 2次フォーム: 詳細（資料請求後の追加情報）
- メールナーチャリングで段階取得

### 6. 質を担保する選び方
リード質を担保するため残すべき項目：
- 業種・従業員規模（BtoB質）
- 役職（決裁権）
- 検討状況（温度感）

## Bad Pattern

- 全項目必須化
- 「住所」を理由なく必須化
- 「FAX番号」（時代）
- 「ご紹介者」（不要）
- 「ニーズ詳細」を100文字以上必須化

## Examples

### Before（必須12項目）
- 会社名、氏名、ふりがな、メール、電話、住所、業種、規模、役職、目的、ご相談、紹介者

### After（必須6項目）
- 会社名、氏名、メール、業種、規模、役職
- 電話・ご相談・紹介者は任意化
- 住所・ふりがなは省略

## Measurement Notes

- フィールド別エラー集計（高エラー = 削減候補）
- マルチステップなら各ステップ完了率
- 必須項目変更前後で A/B テスト

## Checklist

- [ ] 必須項目7以下に削減
- [ ] 質担保項目（規模 / 役職 / 検討状況）を残す
- [ ] 任意項目を明示
- [ ] 自動補完活用
