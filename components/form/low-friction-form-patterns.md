# Low Friction Form Patterns

## Role in LP

フォーム入力負荷を最小化し、送信完了率を最大化する。

## Required Elements

- 必須項目最小限（5〜7個）
- 1カラムレイアウト
- 入力補助
- リアルタイムバリデーション
- 送信フィードバック

## High Conversion Pattern

### 推奨構成（BtoB資料請求）

```
1. 会社名 [必須]
2. 氏名 [必須]
3. メールアドレス [必須]
4. 電話番号 [任意]
5. 役職 [必須] (セレクト)
6. 従業員規模 [必須] (セレクト)
7. ご相談内容 [任意] (テキストエリア)
```

### 推奨構成（採用カジュアル面談）

```
1. 氏名 [必須]
2. メールアドレス [必須]
3. 希望日時（複数選択可） [必須]
4. ひとこと [任意]
```

### 推奨構成（EC購入）

```
1. メールアドレス（ゲスト購入可） [必須]
2. 配送先（住所）
3. 支払方法
4. 規約同意 [必須]
```

## Bad Pattern

- 必須項目10以上
- 2カラムレイアウト（PC）
- 全項目必須
- ハイフン強制
- 送信失敗時の入力消失
- エラー「入力エラー」だけ

## Reduction Tactics

### 項目を減らす
- 「ご相談内容」「興味ある分野」は任意化
- 住所自動補完（郵便番号API）
- 会社名から業種推定

### 入力負荷を下げる
- `inputmode="email"` / `inputmode="tel"`
- `autocomplete="name"` / `autocomplete="email"`
- 半角/全角自動変換
- ハイフン任意

### マルチステップ化
- 項目数が多い場合は分割（10+項目）
- 進捗バー表示
- 戻るボタン明示

## Copy Examples

### 入力補助
- メール: 「半角英数で入力（例: name@example.com）」
- 電話: 「ハイフンの有無は問いません」

### エラー
- 必須未入力: 「[項目名] を入力してください」
- 形式エラー: 「正しい[項目]の形式で入力してください」

### 送信ボタン
- 「無料で資料を受け取る」
- 「3分で資料DL」

### サンクス
- 「お申し込みありがとうございます。
   担当者より3営業日以内にご連絡いたします。」

## Design Notes

- 入力欄高さ48〜56px
- ラベル位置: 上
- フォントサイズ16px以上
- フォーカス時に明示的なハイライト

## Measurement Notes

- 開始計測（focusin）
- エラー計測（invalid イベント）
- 送信計測（submit）
- フィールド別エラー集計（離脱要因特定）
