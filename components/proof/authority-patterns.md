# Authority Patterns

## Role in LP

権威性（受賞 / 認証 / メディア掲載 / 専門家推薦）で信頼を担保する。

## Required Elements

LPに権威要素を最低1〜2種類配置。BtoB / 高単価 / 法規制業界では必須。

## High Conversion Pattern

### 1. 受賞・ランキング
- 「[賞名] 受賞（YYYY年・[主催団体]）」
- 「[ランキング媒体] [カテゴリ] 1位（YYYY年MM月時点）」

### 2. 認証・取得資格
- 「ISO 27001 認証取得」
- 「Pマーク取得」
- 「[業界協会] 加盟」
- 「[業法ライセンス] 取得」

### 3. メディア掲載
- 「[媒体名] に掲載（YYYY年X月号）」
- ロゴと掲載日を併記

### 4. 専門家推薦
- 推薦者氏名・所属・推薦内容
- 利益相反明示（広告・PR の場合）

### 5. 学術エビデンス
- 「[論文] にて[結果]を確認」
- 出典明記

## Bad Pattern

- 「業界最高権威推薦」（誰か不明）
- 受賞年が古すぎ（5年前等）
- 認証バッジ画像のみで詳細なし
- メディア掲載リンクが切れている

## Display Position

- FV: 信頼バッジとして配置
- 戦略中段: 認証セクションで詳述
- 価格直前: 信頼補強として
- フッター: 全権威要素一覧

## Design Notes

- 受賞・認証はロゴ + 詳細
- メディア掲載はロゴ + 掲載日
- 配置は均等・整列

## Measurement Notes

- 認証バッジクリック（詳細ページ）計測

## Legal Notes

- 受賞名は正式名称
- 古い受賞は誤認リスク
- メディア掲載は許諾確認

## Checklist

- [ ] 最低1種類の権威要素
- [ ] 各権威に詳細（年・主催・URL）
- [ ] 古すぎる権威の表記なし
- [ ] 認証は実際に取得済み
