# Case Study Patterns

## Role in LP

ターゲット類似事例を提示し、「自分にも効果がある」と確信させる。

## Required Elements

事例カードに以下を含める：

- 企業ロゴ or 担当者写真（許諾済）
- 業種・規模
- 課題
- 導入の決め手
- 効果（数値）
- 担当者コメント

## High Conversion Pattern

### 推奨配置

- 3〜5件
- 同業種・同規模を優先（共感性）
- 数値効果を必ず含める

### カードレイアウト

```
[ロゴ] [企業名]
業種: ... / 規模: ...

【課題】
...

【導入の決め手】
...

【効果】
- 採用工数: 月150時間 → 月60時間（60%削減）
- 応募数: ...

【担当者コメント】
「導入後3ヶ月で...」
- 担当者名（許諾済）
```

## Bad Pattern

- 「すごく効果がありました」のみ（具体数値なし）
- 数値あるが出典なし（自社調べでも明示必須）
- イニシャル無しで特定可能
- 顧客許諾なき写真使用
- 過度な美化（誇大表現）

## Copy Examples

```
【株式会社 〇〇】 SaaS / 200名

採用業務がスプレッドシートで属人化し、
情報共有に毎週3時間かかっていた。

[サービス名] のAPI連携が決め手で導入。
3ヶ月で採用業務が完全に見える化された。

採用工数: 月150h → 月60h（60%削減）
応募者対応の遅延: 30件/月 → 0件/月

「経営層への報告が、リアルタイムで確認できる
ようになりました」
- 人事部長 [氏名（許諾済）]
```

## Design Notes

- 事例カードは均等サイズ
- ロゴ / 写真を上部に配置
- 数値は太字 or バッジ
- カード全体クリックで詳細ページ（あれば）

## Measurement Notes

- 事例カード閲覧計測
- 詳細ページクリック計測

## Legal Notes

- 顧客の許諾必須
- NDA該当情報は出さない
- 数値の検証可能性を担保

## Checklist

- [ ] 3〜5件配置
- [ ] 同業種・同規模が含まれる
- [ ] 全件に数値効果
- [ ] 顧客許諾確認済
- [ ] 担当者氏名（許諾済）
