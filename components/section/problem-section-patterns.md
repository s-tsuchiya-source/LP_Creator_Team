# Problem Section Patterns

## Role in LP

ターゲットの悩みを言語化し、「自分のことだ」と思わせる。解決策セクションへの橋渡し。

## Required Elements

- 見出し（共感を呼ぶ）
- 悩みリスト 3〜5個
- 共感文（リード or 末尾）

## High Conversion Pattern

### 型A: 「こんな悩みありませんか」型

```markdown
## こんな悩み、ありませんか？

□ 採用業務に月100時間以上を費やしている
□ 応募者管理がスプレッドシートでバラバラ
□ 良い候補者を見落としてしまうことがある
□ 上長への報告に時間がかかる
□ 採用効果の可視化ができていない

ひとつでも当てはまったら、続きをお読みください。
```

### 型B: ストーリー型

```markdown
## 「なんでうちの採用、うまくいかないんだろう」

応募者対応に追われ、本来やるべき面接準備が後回しに。
気づけば、せっかくの優秀候補者を逃してしまう。

そんな経験、ありませんか？
```

### 型C: 数字訴求型

```markdown
## 採用業務に、月100時間使っていませんか？

中小企業の人事担当者の63%が、
採用業務に月50時間以上を費やしているという調査結果（[出典]）。
あなたの貴重な時間を、もっと価値のある業務に使いませんか？
```

## Bad Pattern

- 不安煽り過剰（不安商法と紙一重）
- 解決できない大きな課題を提起（嘘になる）
- 法的問題のある効果効能訴求
- 当たり障りのない「なんとなく不便」

## Design Notes

- チェックリスト風で読みやすく
- 悩みアイコン（必要に応じて）
- 共感文は太字で強調
- 直後に解決策セクションへの誘導

## Measurement Notes

- セクション閲覧時間（Engagement）
- 悩みリストへのスクロール深度

## Industry Notes

### BtoB
- 業務工数・コストの数値化
- KPI への影響

### 採用LP
- 「成長機会がない」「評価されない」等の感情訴求
- ただし誹謗中傷型はNG（前職批判等）

### EC / 物販
- 「失敗した経験」共感
- 効果効能の業法に注意

## Checklist

- [ ] 悩みリスト 3〜5個
- [ ] 数字 or ストーリーで具体化
- [ ] 共感文あり
- [ ] 法務違反なし
- [ ] 不安煽り過剰なし
