# Solution Section Patterns

## Role in LP

問題提起を受けて、自社サービスでどう解決できるかを示す。機能ではなくベネフィットで。

## Required Elements

- 見出し（解決を約束）
- リード文（解決アプローチ全体像）
- 解決アプローチ 3個（機能ではなくベネフィット）

## High Conversion Pattern

### 型A: 3アプローチ型

```markdown
## [サービス名] が解決します

採用業務を一元管理し、求職者対応を自動化することで、
採用工数を最大50%削減します。

### 1. 応募者を一元管理
バラバラな応募経路を一画面で管理。スプレッドシート卒業。

### 2. 候補者へのメッセージを自動化
選考結果の自動返信で、対応漏れゼロ。

### 3. ダッシュボードで効果可視化
経路別CV率・採用単価がリアルタイムで分かる。
```

### 型B: ストーリー型 (Before / After)

```markdown
## 導入前 → 導入後

### 導入前
- 採用業務 月150時間
- 応募者対応の遅延 30件/月
- 経営層への報告が手作業

### 導入後
- 採用業務 月60時間（60%削減）
- 応募者対応の遅延 0件/月
- ダッシュボードで自動レポート
```

### 型C: 機能 → ベネフィット翻訳型

```markdown
## 主要機能とできること

| 機能 | あなたへの効果 |
|---|---|
| API連携 | 既存ツールとの二重入力をなくす |
| 自動メール | 対応漏れゼロ |
| ダッシュボード | 経営層への報告が自動化 |
| 業界テンプレート | すぐに使える、設定不要 |
```

## Bad Pattern

- 機能羅列のみ（ベネフィット欠如）
- 「最高のソリューション」（抽象）
- 効果の根拠なし
- 法的に問題のある効果訴求

## Design Notes

- 各解決アプローチにアイコン
- 番号付き（1, 2, 3）で順序明示
- 数字効果は太字 or バッジ
- スクリーンショット（プロダクト画面）

## Measurement Notes

- セクション閲覧
- 各解決アプローチへのスクロール深度

## Industry Notes

### BtoB SaaS
- 機能 → ベネフィット翻訳必須
- 数字効果（時間 / コスト）

### コンサル
- アプローチ手法を抽象→具体で説明
- 過去の支援実績（数値）

### 採用
- 「働く環境」「成長機会」を具体化
- 抽象訴求NG

## Checklist

- [ ] 3つの解決アプローチ
- [ ] ベネフィット翻訳済み
- [ ] 数字効果（あれば）
- [ ] 法務リスクなし
- [ ] スクリーンショット / 図解
