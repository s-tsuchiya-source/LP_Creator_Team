# Design Differentiation Output Contract

## Purpose

LPのデザイン方針が一辺倒にならず、商材・ターゲット・CV目的に応じて設計されていることを保証する。

## Required Sections

1. Design Concept
2. Reference Direction
3. Icon Strategy
4. UI Component Strategy
5. Typography Strategy
6. Animation Strategy
7. Visual Hierarchy
8. Emotional Role
9. Conversion Role
10. Anti-Generic Design Check
11. CV Impact Hypothesis

## Required Fields

- LP type
- target user
- primary CV
- primary emotion to create
- icon decision and reason
- UI component decision and reason
- typography decision and reason
- animation decision and reason
- visual hierarchy first / second / third
- anti-pattern check result

## Acceptance Criteria

- すべての必須セクションがある
- 各デザイン選択に理由がある
- CVへの貢献仮説がある
- 汎用デザインではないことを説明できる

## Rejection Criteria

- どの商材にも使える説明
- 参考デザインの丸写し
- アイコン・アニメーション・フォントが装飾目的のみ
- CVとの接続がない
