# LP Structure Blueprint Contract

## Purpose

LP構成設計の出力形式を、`lp-structure-blueprint/` の必須項目に基づいて固定化。

## Required Sections

### 1. LPタイプ
- 新規LP / 改善 / 広告 / BtoB / 採用 / SaaS / セミナー / EC / ローカル / 高単価

### 2. 必須構成
- LP Structure Blueprint の 16必須項目（または案件タイプの推奨構成）

### 3. 省略可能構成
- 案件タイプにより省略する項目

### 4. 省略理由
- 各省略項目の理由（lp-director による判断）

### 5. セクション順序
- 完成LP の最終セクション順序

### 6. 感情ジャーニーとの対応
- 9段階感情ジャーニーが各セクションでどう機能するか

### 7. CTA配置
- 配置箇所（最低3箇所）
- 文言
- ハードル段階

### 8. 証拠配置
- 配置箇所
- 証拠種別（数値 / 事例 / 認証 / メディア / お客様の声）

### 9. FAQ配置
- 件数（5〜10件）
- カテゴリ網羅（6カテゴリ）

### 10. 最終CTA
- メインCTA + サブCTA
- 安心要素
- CV後の流れ予告

## Required Fields

- LPタイプ
- 全 16必須項目の含有 / 省略
- 省略理由（省略項目について）
- CTA最低3箇所
- FAQ 5件以上
- 6不安カテゴリ網羅

## Optional Fields

- 業種 playbook 別の特化項目
- A/Bテスト計画

## 4. セクション順序

```

2. 課題提起
3. 解決策
4. 機能・強み
5. 実績・事例
6. 比較表
7. 価格プラン
8. 導入フロー
9. FAQ
10. 最終CTA
```

## Acceptance Criteria

- 16必須項目の含有 / 省略が明示
- 省略項目に理由
- 9段階感情ジャーニー対応
- CTA最低3箇所
- FAQ 5件以上 + 6カテゴリ網羅
- CV後の流れ予告あり

## Rejection Criteria

- 必須項目欠落（理由なし）
- セクション順序断絶
- CTA1箇所のみ
- FAQ 4件以下
- 6カテゴリ未網羅
