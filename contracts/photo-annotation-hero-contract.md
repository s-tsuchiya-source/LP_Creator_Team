# Photo Annotation Hero Contract

## Purpose

photo-annotation-hero パターンの設計結果を構造化された形で記録する。
`outputs/05_design/photo-annotation-hero-plan.md` の必須形式。

## Required Sections

### 1. 写真選定
- 採用写真の概要
- 業務文脈整合
- 採用理由
- 不採用候補と理由

### 2. オーバーレイ設計
- グラデーション or 単色
- 透過率（PC / SP）
- 視認性確保確認

### 3. メインコピー（写真上）
- メインキャッチ（短文 15〜25文字）
- サブコピー（30〜50文字）

### 4. 注釈（callout）
- 配置位置（写真の特定箇所を指す）
- 内容（KPI / 機能名 等）
- 視覚的な指線（line / arrow）

### 5. KPI バッジ
- 配置（コーナー or 端）
- 数値 + 出典

### 6. CTAボタン
- 文言
- 配置位置

### 7. レスポンシブ設計
- PC: 横長写真の活用
- SP: 縦長クロップ + 濃いオーバーレイ

### 8. CV貢献マップ
- 各要素の役割
- ペルソナ視線誘導順

## Format Rules

```markdown
# Photo Annotation Hero Plan

## 1. 写真選定
- 採用写真: 「人事担当者がダッシュボードで状況を確認しているシーン」
- 業務文脈: 採用業務の現場
- 採用理由: ペルソナの業務シーン即時伝達
- 不採用候補: 抽象オフィス風景（業務文脈薄い）/ 笑顔チームミーティング（ストックフォト感）

## 2. オーバーレイ
- 種別: ブランドカラー（紺）グラデーション
- 透過率 PC: 60〜45%（写真が透ける）
- 透過率 SP: 75〜60%（文字視認性優先）
- 視認性: コントラスト 4.5:1 以上確認

## 3. メインコピー
- メインキャッチ: 「採用工数を、最大60%削減」（15文字）
- サブコピー: 「中小企業の人事担当者向け 採用管理ツール」（22文字）

## 4. 注釈
- 配置: 写真右上（ダッシュボード画面を指す）
- 内容: 「ダッシュボードで状況把握 / 月150h → 月60h」
- 指線: 細い線（border 1px solid rgba(255,255,255,0.6)）

## 5. KPI バッジ
- 配置: 写真左下（コーナーバッジ）
- 内容: 「導入1,200社（自社調べ・2026年4月）」
- 内容2: 「ISO 27001 認証」

## 6. CTAボタン
- 文言: 「3分で資料を受け取る」
- 配置: 写真中央〜下部
- 副CTA: 「無料デモを予約」

## 7. レスポンシブ
- PC: 写真横長フル幅 + テキスト左寄せ
- SP: 写真縦長クロップ + テキスト中央集中
- ブレークポイント: 640px

## 8. CV貢献マップ

ペルソナ視線誘導順:
1. ターゲット明示「中小企業の人事担当者へ」（自分ごと化）
2. メインキャッチ「採用工数を、最大60%削減」（メリット）
3. KPIバッジ「導入1,200社」（信頼）
4. 注釈「月150h → 月60h」（期待）
5. CTAボタン（行動）

各要素は CV 直結する位置に配置。
```

## Acceptance Criteria

- 8セクション全て記録
- 写真選定理由 + 不採用候補
- オーバーレイ視認性確保
- 全要素に CV貢献記録

## Rejection Criteria

- ストックフォト採用
- オーバーレイ薄く文字視認性低
- 注釈に意味なし
- KPI 出典なし
