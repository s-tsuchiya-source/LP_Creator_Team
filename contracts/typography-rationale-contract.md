# Typography Rationale Contract

## Purpose

LP内の全テキストラベルの採用理由を記録する。
`outputs/05_design/typography-rationale.md` の必須形式。

## Required Sections

### 1. 抽出した英語ラベル一覧
- 元の英語ラベル
- 配置箇所
- 判定（残す / 日本語化 / 削除）

### 2. 日本語化リスト
- 元 → 日本語化後
- 採用理由

### 3. 削除リスト
- 削除した装飾英語
- 削除理由

### 4. 残した英語リスト
- 残した英語（ブランド名・業界用語等）
- 残した理由

### 5. ラベル文字数チェック
- セクション見出し: 8〜15文字
- ナビ: 4〜8文字
- ボタン: 10〜15文字

### 6. ペルソナ語彙整合
- 業界用語の適切性
- ペルソナ感性に合う

## Required Fields

```yaml
- 元ラベル: FEATURE
- 配置: セクション見出し
- 判定: 日本語化
- 修正後: 3つの特徴
- 採用理由: 「FEATURE」は装飾英語、「3つの」で具体化
- 文字数: 7（適正）
- ペルソナ整合: ✓
```

## Format Rules

```markdown
# Typography Rationale

## サマリー
- 検出英語ラベル: ___ 件
- 日本語化: ___ 件
- 削除: ___ 件
- 残存（理由付き）: ___ 件

## 日本語化リスト

| 元ラベル | 配置 | 修正後 | 採用理由 |
|---|---|---|---|
| FEATURE | セクション見出し | 3つの特徴 | 装飾英語 → 具体化 |
| ABOUT | セクション見出し | 私たちについて | 装飾 → 自然な日本語 |
| FLOW | セクション見出し | 導入の流れ | 意味薄 → 実用語 |
| POINT 01 | カード番号 | 01 | 装飾削除 |
| RECOMMEND | バッジ | おすすめ | 装飾 → 機能語 |

## 削除リスト

| 削除した英語 | 配置 | 削除理由 |
|---|---|---|
| BEAUTY SOLUTION | サブテキスト | 装飾の塊、意味なし |
| SMART | 修飾語 | 抽象すぎ |

## 残した英語リスト

| 残した英語 | 配置 | 残した理由 |
|---|---|---|
| API | 機能説明 | 業界用語、意味あり |
| SaaS | カテゴリ説明 | 業界用語 |
| Notion / Slack | 連携先 | ブランド名 |

## ペルソナ整合チェック

| ラベル | ペルソナ語彙 | 整合 |
|---|---|---|
| 採用工数 | 人事担当者 | ✓ |
| 経路別CV率 | 人事担当者 | ✓（業務語彙） |

## 文字数チェック

| ラベル | 文字数 | 適正範囲 | 判定 |
|---|---|---|---|
| 3つの特徴 | 7 | 8〜15 | やや短いがOK |
| 採用業務を最大60%削減する採用管理ツール | 24 | FV見出しなら可 | OK |
```

## Acceptance Criteria

- 全英語ラベルに判定
- 日本語化したラベルに採用理由
- 残した英語に意味の根拠
- 装飾英語ゼロ
- ペルソナ語彙整合

## Rejection Criteria

- 装飾英語残存
- 採用理由なき英語
- ペルソナ語彙との不整合
- 文字数極端
