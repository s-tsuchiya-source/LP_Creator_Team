# Mandatory Design Differentiation Layer

## Purpose

LP_Creator_Team must not generate one-pattern, generic, AI-like designs. Every LP must choose design elements according to the message, target user, offer, emotional CVR goal, and conversion intent.

## Required References

Use these as design research sources. Do not copy directly. Extract principles, patterns, and component ideas.

- Icons: https://pa-tu.work/tools/icon-search/
- UI Design Dictionary: https://ui-design-dictionary.pages.dev/
- Material Design Components: https://m3.material.io/components
- Curated Web Design References: https://craftwork.design/curated/websites/

## Mandatory Design Decisions

Every LP design brief must include:

1. Design Concept
2. Reference Direction
3. Icon Strategy
4. UI Component Strategy
5. Typography Strategy
6. Animation Strategy
7. Visual Hierarchy
8. Emotional Role
9. Conversion Role
10. Anti-Generic Design Check

## Decision Criteria

### Icon Strategy
Choose line, filled, duotone, pictogram, illustration, realistic visual, product UI capture, or no icon. The choice must match the message and user context.

### UI Component Strategy
Choose components because they help comprehension or conversion, not because they look polished. Consider cards, tabs, accordions, steppers, comparison tables, pricing tables, sticky CTA, modals, forms, timelines, and proof blocks.

### Typography Strategy
Select typography direction based on desired perception: trustworthy, energetic, premium, friendly, technical, editorial, recruitment-oriented, executive, local, or human-centered.

### Animation Strategy
Use motion only when it improves comprehension, expectation, reassurance, or CTA motivation. Avoid decorative motion.

### Visual Hierarchy
Define what users must notice first, second, and third. The hierarchy must guide users toward the main benefit and primary CTA.

### Emotional Role
Design must support at least one emotional role: clarity, expectation, trust, reassurance, urgency, excitement, empathy, or confidence.

### Conversion Role
Design must support at least one conversion role: reduce anxiety, increase motivation, explain mechanism, support proof, simplify comparison, or push CTA.

## Anti-Pattern

Reject outputs that include:

- Generic SaaS blue gradient hero
- Random icons without semantic role
- Same card layout repeated everywhere
- Motion added only for decoration
- Font choice with no message intent
- AI-like abstract illustrations
- Templates that could fit any company
- Beautiful but conversion-irrelevant UI

## Gate Rule

If Design Differentiation is missing, Design Gate and Final Release Gate must fail.
