# final-review-rule.md — 最終レビュー基準

LP公開直前の最終レビュー基準と判定ルール。

## 4観点レビュー

最終レビューは以下4観点で実施：

1. **経営観点** — `lp-executive-reviewer`
2. **CV観点** — `lp-conversion-reviewer`
3. **ブランド観点** — `lp-brand-reviewer`
4. **法務・セキュリティ・QA観点** — `lp-legal-expression-checker` + `lp-security-checker` + `lp-qa-lead`

## 判定区分

各レビュアーは以下の3区分で判定：

- **PASS** — そのまま公開可能
- **NEEDS WORK** — 修正後に公開可能（重要度: BLOCKER / IMPORTANT / NIT）
- **BLOCK** — 公開不可（重大欠陥あり）

## 統合判定（lp-final-gatekeeper）

| 各レビュー結果 | 統合判定 |
|---|---|
| すべて PASS | **公開承認** |
| BLOCK が1つ以上 | **公開不可** — 該当フェーズに差し戻し |
| NEEDS WORK のみ（BLOCKER 含む） | **修正後再判定** |
| NEEDS WORK のみ（IMPORTANT のみ） | 修正推奨 → ユーザー判断で公開可 |
| NEEDS WORK のみ（NIT のみ） | **公開承認** — 修正は任意 |

## 各観点の判定基準

### 経営観点
- [ ] 事業KPIに直結
- [ ] 売上 / 利益貢献仮説が現実的
- [ ] ブランド戦略整合
- [ ] 投資対効果が説明可能

### CV観点
- [ ] 訴求軸 → CTA → フォーム の一貫性
- [ ] 不必要な離脱要因なし
- [ ] フォーム入力負荷最小
- [ ] CTA配置・文言の最適化

### ブランド観点
- [ ] ブランドガイドライン準拠
- [ ] トーン整合
- [ ] ビジュアル整合
- [ ] 信頼性表現

### 法務・セキュリティ・QA
- [ ] 誇大表現なし
- [ ] No.1表現の根拠あり
- [ ] 個人情報取扱が法令準拠
- [ ] HTTPS / プライバシーポリシー
- [ ] 主要ブラウザ動作OK
- [ ] レスポンシブOK
- [ ] フォーム送信OK
- [ ] 計測動作OK

## 差し戻し時の対応

1. BLOCK 理由を該当フェーズに通知
2. 修正範囲を明示
3. 修正後、該当エージェントが再判定
4. PASS 後に最終承認を再実行

## 公開後監視

公開後、以下を監視：

- [ ] CV発生（最初の1日 / 1週間）
- [ ] エラー発生
- [ ] フォーム送信成功率
- [ ] 計測の動作
- [ ] 実機でのバグ報告

異常があれば即時ロールバック（手順は `lp-release-manager` の指示書に従う）。
