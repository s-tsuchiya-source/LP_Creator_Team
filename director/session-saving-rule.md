# session-saving-rule.md — セッション節約ルール

Claude Code のセッション消費を抑えるための運用ルール。

## 原則

1. **全エージェントを使わない** — 必要最小限のみ
2. **重複レビューを避ける** — 同一観点を複数エージェントで重複させない
3. **まずディレクターが仮説を立てる** — エージェント呼び出し前に仮説で前進
4. **エージェント呼び出しは目的を明確にする** — 曖昧な依頼で往復を増やさない
5. **中間成果物を冗長化しない** — 必要十分の粒度
6. **最終出力に必要な情報だけ統合する** — 中間ノイズを削ぎ落とす
7. **不明点は仮説補完する** — 確認は重要不明点のみ
8. **重要不明点だけ確認する** — CV / ターゲット / 商材 / 法務

## 実務テクニック

### 仮説先行
```
[NG] エージェントに丸投げ → 中間成果物 → 修正指示 → 再生成
[OK] ディレクターが仮説 → エージェントに「この仮説の検証と精緻化」依頼 → 1回で着地
```

### バッチ呼び出し
独立した複数エージェントは並行呼び出し可能：
- `lp-persona-researcher` + `lp-competitor-researcher` + `lp-market-researcher` を同時呼び出し

### 共有コンテキストの活用
- ペルソナ・戦略は1回作成し、後段エージェントは読むだけ
- 同じ情報を再生成させない

### 中間成果物の集約
- 各エージェント中間成果物は `outputs/` に保存し、次エージェントは読み込む
- 会話文脈に貼り付け続けない

### スキップ判定
- 簡易LPでは戦略・リサーチ・レビューボードをスキップ可
- 既存改善ではフル戦略フェーズをスキップ可

## 禁止パターン

- 全43エージェントを順次呼び出す
- 同じ観点を複数エージェントに重複依頼（例: コピー品質を複数レビュアーへ）
- 中間成果物を毎回再生成
- 12問を超える初回ヒアリング
- 重要不明点でない箇所を何度も確認

## 推奨パターン

- 案件タイプを最初に判別 → 最小エージェント構成を決定
- 仮説 → 検証 → 確定 の3ステップで前進
- レビューは最終承認エージェントに集約
- ユーザーへの中間報告は要点のみ（フェーズ完了時 1〜2 回）

## チェック

エージェントを呼び出す前に、以下を確認：

- [ ] このエージェントの成果物がないと最終出力できないか
- [ ] 他エージェントで代替できないか
- [ ] 仮置き補完で前進できないか
- [ ] 入力情報は揃っているか

すべて Yes でなければ呼び出しを再考する。
