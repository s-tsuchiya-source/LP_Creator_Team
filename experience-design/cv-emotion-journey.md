# CV感情ジャーニー

LP閲覧からCVまでの感情変化を、9段階で設計します。

## 9段階の感情ジャーニー

```
[1. 流入]
   ↓
[2. 自分ごと化]
   ↓
[3. 課題共感]
   ↓
[4. 期待]
   ↓
[5. 信頼]
   ↓
[6. 不安解消]
   ↓
[7. 行動意欲]
   ↓
[8. CV直前の決断]
   ↓
[9. CV後の安心]
```

## 各段階の心理と対応LP要素

### 1. 流入（0〜2秒）

**心理**: 「これ何のサービス？」「自分に関係ある？」

**LP要素**: 
- ファーストビュー（メインキャッチ + ベネフィット）
- 信頼バッジ
- ターゲット明示（「中小企業の人事担当者向け」など）

**設計目標**: 3秒で「自分のための情報か」判断できる

### 2. 自分ごと化（5〜15秒）

**心理**: 「これ、自分の課題かもしれない」

**LP要素**:
- 「こんな悩みありませんか」セクション
- 業種・役職別ペルソナ訴求
- チェックリスト形式の悩み列挙

**設計目標**: ユーザーが「自分のことだ」と確信する

### 3. 課題共感（15〜30秒）

**心理**: 「分かってくれている」「これは深刻な問題かも」

**LP要素**:
- 課題提起セクション（言語化されていない悩みを言語化）
- 放置した未来（暗示）
- 数字で課題の大きさを示す

**設計目標**: 課題への意識を高める

### 4. 期待（30秒〜1分）

**心理**: 「これなら解決できそう」「自分の業務がこう変わる」

**LP要素**:
- 解決策セクション（機能 → ベネフィット翻訳）
- 主要機能 × ベネフィット
- 導入後の未来像（時系列）
- 数値効果（出典付き）

**設計目標**: 期待感を高める（誇張せず）

### 5. 信頼（1〜2分）

**心理**: 「本当にできるんだろうか」「信頼できる会社？」

**LP要素**:
- 導入企業ロゴ列
- 数値実績（出典付き）
- 同業種事例
- 認証バッジ（ISO / Pマーク）
- メディア掲載
- お客様の声（許諾済）

**設計目標**: 「実在の会社が、実在の成果を出している」と確信させる

### 6. 不安解消（2〜3分）

**心理**: 「でも〇〇が心配」「失敗したくない」

**LP要素**:
- FAQ（5〜10件）
- 比較表
- 導入フロー（時間明示）
- サポート体制
- 解約条件
- セキュリティ情報

**設計目標**: 主要な不安カテゴリを全て解消

### 7. 行動意欲（3〜4分）

**心理**: 「いま行動するメリット」「資料くらい見てみるか」

**LP要素**:
- オファー再訴求
- 限定性 / 緊急性（実態に即す）
- 社内検討資料（BtoB）
- ROI試算（BtoB）

**設計目標**: 「今動く理由」を明確化

### 8. CV直前の決断（4〜5分）

**心理**: 「申し込んでから何が起きる？」「営業されすぎない？」

**LP要素**:
- 最終CTAセクション
- 「営業はご希望の方のみ」明記
- フォーム項目を最小限に
- プライバシーポリシーリンク
- 自動返信メールの予告

**設計目標**: CV直前の心理ハードルをゼロにする

### 9. CV後の安心（CV後）

**心理**: 「無事申し込みできた」「次は何が起きる？」

**LP要素**:
- サンクスページ
  - 「申し込みありがとうございます」
  - 「3営業日以内に担当者よりご連絡」
  - 自動返信メールの説明
  - 関連資料DLリンク

**設計目標**: CV後の不安をゼロにし、商談化率を高める

## 各段階の感情と LP 要素マトリクス

| 段階 | 必要な感情 | LP要素 |
|---|---|---|
| 1. 流入 | 興味 | FV / 信頼バッジ |
| 2. 自分ごと化 | 共感 | 悩みリスト |
| 3. 課題共感 | 緊張 | 問題提起 / 暗示 |
| 4. 期待 | わくわく | 解決策 / 未来像 |
| 5. 信頼 | 安心 | 実績 / 事例 / 認証 |
| 6. 不安解消 | 確信 | FAQ / 比較 / フロー |
| 7. 行動意欲 | 推進力 | オファー / 限定性 |
| 8. CV直前 | 決断 | 最終CTA / ハードル下げ |
| 9. CV後 | 安心 | サンクスページ |

## ジャーニーが破綻するパターン

### NG パターン1: 期待だけが先行
- FVで期待を煽る
- 信頼要素・不安解消なし
- → CVするが、商談化率が低い（実態とのギャップ）

### NG パターン2: 信頼要素ばかり
- 認証・受賞・実績ばかり
- ベネフィット・期待・自分ごと化が薄い
- → 「すごい会社だな」で終わる

### NG パターン3: 機能羅列
- ベネフィット翻訳なし
- 感情ジャーニーが断絶
- → 読み飛ばされる

### NG パターン4: 不安解消なし
- 信頼・期待は十分
- でもFAQ・比較表・解約条件が不明
- → CV直前で迷って離脱

### NG パターン5: CV後が不明
- フォーム送信完了後の流れが書いてない
- → 「申し込んだけど何が起きる？」で不安発生

## 設計手順

1. ペルソナの動機マップを作成（`user-motivation-map.md`）
2. 9段階のジャーニーを描く
3. 各段階のLP要素を設計
4. セクション構成にジャーニーを反映
5. 各段階の感情変化を レビュー（lp-final-gatekeeper）
