# 感情とCVの原則

LPにおいて、CV（コンバージョン）は論理だけでは起きません。
ユーザーは情報を理解した後、感情で動きます。

このファイルは、Emotional CVR Layer の最上位原則をまとめたものです。

## 原則1: CVは論理だけでは起きない

### 論理だけのLP（CVRが伸びにくい）

- 機能羅列
- スペック一覧
- 「業界初」「最高品質」だけ
- 数字を出すが意味が伝わらない

### 論理 + 感情で動くLP

- 機能を「あなたが楽になる理由」に翻訳
- スペックを「課題が解決する根拠」に変換
- 数字を「導入後の未来像」に接続
- 「自分のことだ」と感じる構成

## 原則2: ユーザーはメリットを理解した後、期待・安心・納得で行動する

行動前のユーザー心理は、3段階で進みます。

```
理解 → 期待 → 安心 → 納得 → 行動
```

各段階で必要なLP要素：

| 段階 | ユーザー心理 | LP要素 |
|---|---|---|
| 理解 | 何ができるサービスか分かる | FV / 解決策セクション |
| 期待 | 自分の課題が解決しそう | ベネフィット / 未来像 |
| 安心 | 失敗しなさそう | 実績 / 事例 / 保証 |
| 納得 | これを選ぶ理由が分かる | 比較 / 差別化 |
| 行動 | 押す理由がある | CTA直前 / オファー / フロー |

## 原則3: BtoB でも感情は重要

BtoBは「合理判断」と思われがちですが、実際は以下の感情で動きます。

### BtoB ユーザーが求める感情

- **「成果が出そう」** — 期待
- **「失敗しなさそう」** — 安心
- **「社内説明できそう」** — 自信
- **「上司に推せる」** — 推進力
- **「比較が楽になりそう」** — 効率
- **「営業に押し売りされなさそう」** — 安心

「楽しそう」「わくわくする」だけでは BtoB のCVは取れません。
**期待・安心・納得・推進力**を作る感情設計が必要です。

## 原則4: BtoC は「変われそう」を作る

BtoCユーザーの感情：

- **「使ってみたい」** — 興味
- **「変われそう」** — 期待
- **「今より良くなりそう」** — 希望
- **「自分にもできそう」** — 自信
- **「失敗しても大丈夫そう」** — 安心

具体的な未来像（Before/After / 体験談）で感情を動かします。

## 原則5: 採用LP は「ここで働く自分」を想像できることが鍵

採用LPの感情設計：

- **「自分のスキルが活きそう」** — 期待
- **「成長できそう」** — 希望
- **「カルチャーが合いそう」** — 安心
- **「メンバーと働きたい」** — 親近感
- **「給与・働き方も納得」** — 合理性

抽象的な「成長できます」「やりがい」では刺さりません。
**具体的な日常・社員ストーリー・制度**で「働く自分」を想像させます。

## 原則6: 期待感は誇張ではなく、具体的な未来像で作る

期待感を作る方法は2種類あります。

### NG: 誇張で作る期待
- 「絶対に成功します」
- 「100%成果保証」
- 「業界の常識を変えます」（根拠なし）
- 派手な装飾でわくわく感を演出

→ 短期的にCVが上がっても、長期的に信頼を毀損します。

### OK: 具体性で作る期待
- 「導入後3ヶ月で、採用工数が60%削減（自社調べ・100社平均）」
- 「初期設定は専任担当が支援。最短2週間で稼働」
- 「導入企業の85%が継続利用（自社調べ・2025年実績）」
- お客様の声で「導入後の実際の変化」を伝える

→ 信頼を保ちながら期待を高めます。

## 原則7: 感情設計は全フェーズに通す

Emotional CVR は、コピーやデザインだけの仕事ではありません。

すべてのフェーズで感情設計が通っているか確認します。

```
戦略フェーズ: ターゲットの感情ニーズを把握
構成フェーズ: 感情ジャーニーをセクションで設計
コピーフェーズ: ベネフィット・期待・未来を言語化
デザインフェーズ: 信頼・親しみ・期待を視覚化
実装フェーズ: フォーム入力の心理負荷を下げる
計測フェーズ: 感情変化のポイントを可視化
レビューフェーズ: 感情ジャーニーが破綻していないか確認
```
