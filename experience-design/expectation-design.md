# 期待感設計

期待感は、ユーザーがCVに踏み切る最後の一押しです。
ただし、誇張・煽りで作ると信頼を毀損します。

このファイルは、**根拠ある期待感**の作り方を整理します。

## 期待感とは何か

期待感 =「この先、自分にとって良いことが起きる予感」

3要素：
1. **可能性の認識**: 「自分の課題が解決できそう」
2. **具体性の理解**: 「どう変わるか想像できる」
3. **実現性への確信**: 「この会社/サービスならできそう」

## 誇大表現との違い

### 誇大（NG）
- 「絶対に成功します」
- 「業界の常識を覆します」（根拠なし）
- 「100%効果を保証」
- 「導入翌日から売上倍増」
- 抽象的な「最高」「最強」

→ 一時的な期待は作れるが、信頼を失う / 法務リスク

### 根拠ある期待（OK）
- 「導入後3ヶ月で、採用工数が平均60%削減（自社調べ・100社）」
- 「最短2週間で導入完了。専任担当が支援」
- 「導入企業の85%が継続利用（2025年実績）」
- 「あなたの業界では、こんな課題解決事例があります」

→ 信頼を保ちながら期待を高める

## 根拠の作り方（4種）

### 1. 数値根拠
- 利用社数 / 導入数
- 改善率（時間 / コスト / 売上）
- 継続率 / 満足度
- すべて出典付きで

### 2. 事例根拠
- 同業種・同規模の事例
- Before / After の具体性
- 担当者コメント（許諾済）

### 3. プロセス根拠
- 導入フロー（ステップ別 + 所要時間）
- サポート体制
- 担当者紹介

### 4. 第三者根拠
- 受賞・認証
- メディア掲載
- 専門家推薦

## 導入後の変化を具体化する

「導入したらどう変わるか」を時系列で描きます。

### テンプレート

```
【導入前】
- 業務工数: 月150時間
- 報告書作成: 週8時間
- データ抜け漏れ: 月10件以上

【導入1ヶ月後】
- 操作に慣れ、初期効果を実感

【導入3ヶ月後】
- 業務工数: 月60時間（60%削減）
- 報告書: 自動生成で実質ゼロ
- 抜け漏れ: ほぼゼロ

【導入6ヶ月後】
- 経営判断のスピード向上
- 上司から「楽になった」と評価
- 部門間連携の改善

※ 自社調べ・100社平均（2026年4月時点）
```

## 期待感の「適温」を保つ

期待を高めすぎると、不信感を生みます。

### 適温の作り方

- **約束しすぎない**: 「絶対」は使わない
- **「あなたの業界なら」と限定する**: 普遍化しない
- **ばらつきを認める**: 「個人差・業種差があります」明記
- **失敗事例も触れる**: 「合わなかった場合の対応」もFAQに

### 期待を下げないけど、信頼を保つ表現

- 「多くの場合は〜」
- 「業界平均では〜」
- 「同規模企業では〜」
- 「貴社の状況に合わせて〜」

## 「まずは資料を見てみたい」と思わせる期待の作り方

CV直前で必要な期待は、**「まず情報を見たい」程度のハードル低い期待**です。

### CV直前のコピー例

```
資料には、以下が書かれています：

- 同業種事例 5社
- 機能一覧と業務適用例
- 価格・プラン詳細
- 導入支援の流れ
- セキュリティ情報

3分で資料が届きます。
営業からの連絡は、ご希望の方のみです。

[無料で資料を受け取る]
```

→ 「資料を見るだけ」のハードルを下げ、不安要素（営業押し売り）を解消し、期待（具体的な情報が得られる）を作る

## ターゲット別の期待ポイント

### BtoB
- 成果が出る期待
- 失敗しない期待
- 社内説明できる期待
- 業務が楽になる期待

### BtoC
- 変われる期待
- 楽しめる期待
- 失敗しても保証される期待

### 採用LP
- 成長できる期待
- カルチャーフィットする期待
- メンバーと働ける期待
- 給与・働き方が良くなる期待
