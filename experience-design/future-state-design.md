# 未来像設計

ユーザーが「導入後・利用後の自分」を具体的に想像できるかが、CVに直結します。

## 4つの未来像

LP内では、4つの未来をユーザーに描いてもらいます。

### 1. 現状の不満（暗示）
- 今のまま続けるとどうなるか
- 課題が放置された未来
- 競合に置いていかれる未来

### 2. 放置した場合の未来
- 課題が大きくなる
- コスト増 / 機会損失
- 担当者の疲弊

### 3. 導入後の未来
- 短期的な変化（〜1ヶ月）
- 中期的な変化（〜3ヶ月）
- 長期的な変化（半年〜）

### 4. 成果が出た後の未来
- 業務が回る理想状態
- 上司・経営層からの評価
- 自分自身のキャリア向上
- 求職者なら「働く自分」のイメージ

## 業務上の未来像（BtoB）

### 例: 採用管理SaaS

```
【現状の不満】
- スプレッドシートで応募者管理
- 抜け漏れが頻発
- 経営報告に毎週8時間

【放置した未来】
- 応募者対応の遅延 → 内定辞退増
- 業務疲弊 → 担当者退職
- 経営層への不信 → 予算削減

【導入1ヶ月後】
- 応募者一覧がダッシュボード化
- 操作に慣れ、初期効果実感

【導入3ヶ月後】
- 採用工数 60%削減
- 抜け漏れゼロ
- 自動レポート生成

【成果が出た後】
- 経営層が状況をリアルタイム把握
- 採用部門の評価向上
- 担当者は本来の業務に集中
- 「あなたが導入したから」と評価される
```

## 心理的・キャリア的な未来像

業務効率化だけでなく、「自分の心理・キャリア」がどう変わるかも描きます。

### 心理面
- 「もう手作業で疲弊しなくていい」
- 「ミスを心配しなくていい」
- 「上司に怒られない」
- 「胸を張って報告できる」

### キャリア面
- 「上司から信頼される」
- 「次のポジションが見える」
- 「業界での市場価値が上がる」
- 「転職時にもアピールできる」

## 採用LP の「働く未来」

求職者にとっては、「ここで働く自分」が想像できることが最重要です。

### 描くべき未来

#### 1日の流れ
```
9:00 出社（リモートも可）
9:30 朝会で当日タスク確認
10:00 開発に集中
12:00 ランチ（メンバーと）
13:00 ペアプロ・コードレビュー
15:00 1on1（マネージャーと）
17:00 進捗共有
18:00 退社
```

#### 1ヶ月後
- 業務に慣れ、独自タスクが進む
- メンバーとの関係構築

#### 3ヶ月後
- 主要機能の改善を担当
- 技術スタックに習熟

#### 1年後
- リーダーポジションへステップアップ
- 採用面接への参加
- 給与改定

## BtoC の未来像

### 例: スクール / 教育

```
【現状】
独学で挫折、転職できないまま

【放置した未来】
スキル不足のまま、評価されない

【受講開始後】
週末にコツコツ学習、モチベーション維持

【3ヶ月後】
ポートフォリオ完成、面接の自信

【6ヶ月後】
転職成功、年収50〜100万円アップ

【1年後】
キャリア選択肢の拡大
```

## 未来像をセクションに落とす方法

### セクション例: 「導入後の流れ」

```markdown
## 導入後、こう変わります

### 1ヶ月目
業務に慣れ、初期効果を実感
- 業務工数: 約20%削減

### 3ヶ月目
本格的な効果が見え始める
- 業務工数: 約60%削減（自社調べ・100社平均）
- 抜け漏れ: ほぼゼロ

### 6ヶ月目
組織全体での効果実感
- 経営層への報告自動化
- 部門間連携の改善

※ 効果には個人差・業種差があります
```

### セクション例: 「あなたの未来」

```markdown
## あなたの業務が、こう変わります

### 朝
- 出社時に、ダッシュボードで状況把握
- 「あ、ここ対応済みだな」と確認するだけ

### 日中
- 重要な意思決定に集中できる
- 手作業のデータ集計はゼロ

### 夕方
- 経営層への報告は、自動レポートをチェックするだけ
- 残業せずに帰れる

### 週末
- 業務を持ち帰らず、休める
```

## 未来像の質を高めるコツ

### 1. 抽象的にしない
- ❌「効率化できる」
- ✅「月150時間 → 月60時間（60%削減）」

### 2. 一人称にする
- ❌「業務が改善されます」
- ✅「あなたは手作業から解放されます」

### 3. 五感に訴える
- ❌「楽になる」
- ✅「朝、ダッシュボードを見るだけで状況が把握できる」

### 4. 周囲の反応も描く
- ❌「効果が出る」
- ✅「上司から『楽になったね』と評価される」

### 5. 出典・条件を明示
- 数値には必ず出典
- 「個人差・業種差があります」明記
