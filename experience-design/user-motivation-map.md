# ユーザー動機マップ

ユーザーの行動動機を、顕在・潜在・感情・合理の4軸で整理します。

## 動機の4分類

### 1. 顕在ニーズ
ユーザー自身が言語化できているニーズ。

例（採用管理SaaS）:
- 「採用業務を効率化したい」
- 「応募者管理を楽にしたい」
- 「報告書作成の時間を減らしたい」

LP上では、課題セクションで言語化する。

### 2. 潜在ニーズ
ユーザー自身は気づいていないが、確かにあるニーズ。

例:
- 「上司から『分かりやすい』と評価されたい」
- 「採用の質を可視化したい」
- 「他部門との連携をスムーズにしたい」

LP上では、強みセクションや事例セクションで「気づき」を提供する。

### 3. 感情的ニーズ
論理ではなく感情で動くニーズ。

例:
- 「失敗したくない」
- 「上司に怒られたくない」
- 「業務疲弊から解放されたい」
- 「胸を張って報告したい」

LP上では、共感セクション・FAQで対応する。

### 4. 合理的ニーズ
データや論理で判断するニーズ。

例:
- 「ROIが説明できる」
- 「セキュリティ要件を満たす」
- 「既存システムと連携できる」

LP上では、実績・比較表・セキュリティセクションで対応する。

## 緊急性の3レベル

### 高緊急性
- 「今すぐ解決しないと業務が回らない」
- 「期日があり、間に合わない可能性」
- 「競合が動き始めている」

LP上では、限定性・期限を提示。

### 中緊急性
- 「半年〜1年以内に解決したい」
- 「今期の予算内で進めたい」
- 「組織変更に合わせたい」

LP上では、ROI・スケジュール提示。

### 低緊急性
- 「いつかは解決したい」
- 「情報収集段階」
- 「比較検討中」

LP上では、資料DL・無料相談などローハードルCTAを用意。

## 行動阻害要因（CV阻害）

ユーザーがCVを止める理由：

### 1. 価格不安
- 予算が見えない
- ROI が説明できない

### 2. 導入不安
- 時間がかかりそう
- 現場が反対しそう

### 3. 効果不安
- 同業種事例がない
- 数値根拠が弱い

### 4. 信頼不安
- 会社情報が薄い
- セキュリティが不明

### 5. 心理的負担
- フォーム項目が多い
- 営業の押し売り感
- 個人情報の取扱不安

### 6. 比較疲れ
- 他社と何が違うか不明
- 比較表がない

## 行動促進要因（CV促進）

ユーザーをCVに進める要因：

### 1. 自分ごと化
- 「これ、自分のためのサービスだ」と感じる

### 2. 期待感
- 「自分の課題が解決しそう」と思える

### 3. 安心感
- 「失敗しないと思える」根拠がある

### 4. 推進力
- 「上司を説得できる」素材がある

### 5. 緊急性（実態に即す）
- 「今動く理由」が分かる

### 6. ハードル低い行動
- 「資料DLだけでも」と思える

### 7. CV後の安心
- 「申し込んだ後の流れ」が見える

## CV直前の心理状態

CV直前のユーザーの頭の中：

```
・「資料を見るだけなら…」
・「営業されすぎないかな」
・「個人情報の扱いは大丈夫？」
・「フォーム入力、面倒くさそう」
・「申し込んでから何が起きる？」
・「上司にどう説明しよう」
```

これらに対応するLP要素：

| 心理 | LP要素 |
|---|---|
| 資料を見るだけなら | 「資料DLのみ可」明示 |
| 営業されすぎないか | 「営業はご希望の方のみ」 |
| 個人情報は？ | プライバシーポリシーリンク |
| フォーム面倒 | 必須項目最小限 |
| 申込後何が起きる？ | 流れ提示 |
| 社内説明 | 比較資料・ROI試算同梱 |

## 動機マップを LP に反映する手順

1. ペルソナを精査
2. 4分類（顕在 / 潜在 / 感情 / 合理）でニーズを整理
3. 緊急性レベルを判定
4. 阻害要因を10個以上リストアップ
5. 促進要因を LP の各セクションに配置
6. CV直前の心理対応を最終CTAに反映

## ターゲット別動機例

### BtoB SaaS（採用管理）
- 顕在: 採用工数削減
- 潜在: 経営層への報告自動化
- 感情: 「業務疲弊から解放されたい」
- 合理: ROI / セキュリティ
- 緊急性: 中（今期予算で進めたい）
- 阻害: 社内説明・運用負荷
- 促進: 同業種事例 / 比較表 / 専任サポート

### 採用LP（エンジニア）
- 顕在: 転職して年収アップ
- 潜在: スキルが活きる仕事 / 評価される
- 感情: 「今の会社に閉塞感」
- 合理: 年収・働き方・技術スタック
- 緊急性: 中〜低（1年以内）
- 阻害: カルチャー不安 / 失敗リスク
- 促進: 社員ストーリー / カジュアル面談 / 評価制度

### EC（化粧品）
- 顕在: 肌悩み解消
- 潜在: 自信を取り戻す
- 感情: 「失敗したくない」
- 合理: 成分 / 価格 / 返金保証
- 緊急性: 高（即解決したい）
- 阻害: 副作用不安 / 高い
- 促進: 30日返金 / 初回割引 / 成分表示
