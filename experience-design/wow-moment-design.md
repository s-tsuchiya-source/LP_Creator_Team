# WOW モーメント設計

LP内で「おっ」と思わせる瞬間を作ります。
ただし、**派手な演出ではなく、納得・安心・共感のWOW**を重視します。

## WOWの誤解

### NG: 派手な演出のWOW
- 大きなアニメーション
- 派手なグラデーション
- 動画の自動再生（音付き）
- カウントダウンタイマー（嘘の限定性）

→ 一時的に注意を引くが、信頼を毀損する。
特にBtoBではブランド毀損リスク。

### OK: 納得・共感のWOW
- 「自分の悩みを言語化してくれた」
- 「思っていたより分かりやすい」
- 「そんな解決策があったのか」
- 「ここまでサポートしてくれるのか」
- 「これなら社内で説明しやすい」

## 6種類のWOWパターン

### 1. 「自分の悩みを分かってくれている」WOW

ユーザーが言語化できていない悩みを、LPが先に言語化する。

**例（BtoB SaaS 採用管理）**
```
こんな経験、ありませんか？

□ 月曜の朝、応募者一覧のスプレッドシートを開くのが憂鬱
□ 「あの応募者、返信した？」と上司に聞かれるのが怖い
□ 良い候補者を見落としていないか、夜眠れない時がある

このLPは、そんな採用担当者のために作られました。
```

→ ユーザー：「これ、自分のことだ」

### 2. 「思っていたより分かりやすい」WOW

業界用語や複雑な機能を、シンプルに説明する。

**例**
```
APIだ、データ連携だと言われても、難しく感じますよね。

簡単に言うと、こうです：

「今使ってるツール（Excel / Salesforce / 既存ATS）」と
[サービス名] を、自動でつなぐ機能です。

二重入力がなくなり、月X時間削減できます。

[3分で資料DL]
```

→ ユーザー：「あ、シンプルだ」

### 3. 「そんな解決策があったのか」WOW

業界の暗黙のやり方をひっくり返す視点を提示する。

**例（採用LP）**
```
「採用は数を打って当てる」、その常識は本当ですか？

応募者を闇雲に増やすより、
1人の質を高める方が、結果的に採用効率が3倍になる
というデータがあります。

[サービス名] は、量より質に振り切った採用支援です。
```

→ ユーザー：「視点が違う、興味深い」

### 4. 「ここまでサポートしてくれるのか」WOW

サポートの手厚さを具体的に伝える。

**例**
```
[サービス名] では、導入後に以下を全て行います：

✓ 専任担当による設定支援（無料）
✓ スタッフ向け研修（オンライン）
✓ データ移行の代行（無料）
✓ 月次レビュー（任意）
✓ 24時間チャットサポート

「導入後に放置される」を絶対にしません。
```

→ ユーザー：「これなら安心して始められる」

### 5. 「これなら社内で説明しやすい」WOW

BtoBで決定的に重要なWOW。「上司に推せる素材」を提供する。

**例**
```
社内検討で使える資料を、無料でご用意しています：

✓ 機能比較表（競合A社・B社 vs 自社）
✓ ROI試算シート（あなたの会社規模で）
✓ 導入企業の事例集（業界別）
✓ セキュリティチェック資料（情シス向け）

[3分で資料パックを受け取る]
```

→ ユーザー：「これは上司に推しやすい」

### 6. 「まず資料だけ見てみたい」WOW

CV直前に、心理的ハードルを下げる。

**例**
```
資料だけご覧いただいて、判断ください。

✓ 営業からの強引な連絡はありません
✓ ご希望の方のみ、個別相談を行います
✓ 解約はいつでも可能です（契約後）

3分で資料が届きます。

[無料で資料を受け取る]
```

→ ユーザー：「これなら気軽に申し込める」

## WOWの配置

LP内のどこにWOWを配置するか：

| 位置 | WOWの種類 |
|---|---|
| FV | 自分ごと化（型1） |
| 課題セクション | 悩みの言語化（型1） |
| 解決策セクション | 視点の転換（型3） |
| 機能セクション | 分かりやすさ（型2） |
| サポートセクション | 手厚さ（型4） |
| 社内検討セクション | 推し材料（型5） |
| CTA直前 | ハードル下げ（型6） |

## WOWの作り方プロセス

1. ペルソナの「言語化されていない悩み」を10個リストアップ
2. その中から「LP内で言語化できるもの」を3〜5個選ぶ
3. 各セクションに WOW タイプを割り当てる
4. コピー化する
5. デザイナーと連携し、視覚化する（過剰演出は避ける）

## WOW の質を保つルール

### やってはいけない

- 嘘の限定性（常時セール表記）
- 誇大な数字（根拠なし）
- 派手な装飾だけのWOW
- 不安を過剰に煽るWOW
- 競合への一方的批判

### やるべき

- 「事実」+「視点」で WOW を作る
- 出典・根拠付き
- 信頼を毀損しない範囲
- ユーザーが共感できる粒度

## チェックリスト

- [ ] LP内に最低3つのWOWを配置
- [ ] 派手な演出だけでWOWを作っていない
- [ ] 「自分ごと化」のWOW（型1）がある
- [ ] 「分かりやすさ」のWOW（型2）がある
- [ ] BtoBなら「社内検討」のWOW（型5）がある
- [ ] CV直前のハードル下げWOW（型6）がある
- [ ] 嘘の限定性・誇大表現なし
