# Problem Visualization Hero

## 概要

ユーザーが抱える「課題」をインフォグラフィック・図解で視覚化するパターン。

## Best Use Cases

- 課題が抽象的で言語化困難な商材
- 「なぜ今やるべきか」を視覚で伝える
- データ駆動の意思決定者向け
- BtoB 経営判断者向け商材

## NG Use Cases

- 課題が単純すぎる商材
- 視覚化しても伝わらない感情系商材

## 構成要素

```
- 課題を表す統計・グラフ
  - 業界平均データ
  - 業務工数の現状
  - 競合差・改善余地
- 数字のインパクト
- メインキャッチ「この状態、続けますか？」型
- メインCTA
```

## CV貢献

- 課題の重要性即時理解
- 「自分の業界もそうだ」共感
- データ駆動の説得力

## 例

```
[統計図解]
  円グラフ:
    - 採用業務に追われる時間: 60%
    - 戦略業務: 25%
    - その他: 15%

  注釈: 「採用担当者の業務時間配分（業界平均・自社調べ）」

  メインキャッチ:
  「業務時間の60%が、採用業務の手作業に消えている。」

  サブコピー: 「あなたの会社も、こうなっていませんか？」

  CTA: 「あなたの業務工数を診断する」
```

## 注意点

- 統計の出典明記
- 「業界平均」表現は根拠付き
- 不安煽り過剰NG

## 採用判断

```
文脈: 5
共感: 5（業界共通課題）
CV: 5（強い動機付け）
差別化: 5（独自視覚化）
実現: 3（インフォグラフィック工数）
合計: 23
```
