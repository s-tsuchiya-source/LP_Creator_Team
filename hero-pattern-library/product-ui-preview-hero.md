# Product UI Preview Hero

## 概要

プロダクト画面・ダッシュボードのスクリーンショットをヒーローに配置するパターン。

## Best Use Cases

- 商材の利用シーンが画面操作で完結する場合
- ペルソナがプロダクト画面で意思決定する場合
- 画面UI が業界比較で優れている場合

## NG Use Cases

- 「BtoB SaaS だから」の理由だけでデフォルト採用
- 画面UI が業界平均レベル
- 商材が画面操作以外（コンサル / 研修 等）

## 構成要素

```
- ヘッダー（ロゴ + ナビ）
- 左: テキスト
  - ターゲット明示
  - メインキャッチ
  - サブコピー
  - メインCTA + 副CTA
- 右: プロダクト画面 SS
  - 主要機能を見せる
  - 注釈で要素を強調
- 下: 信頼バッジ・ロゴ列
```

## CV貢献

- 利用イメージの即時伝達
- 機能の具体性
- プロダクト品質の信頼

## 注意点

- 単純な画面 SS は「装飾」になりがち
- 必ず注釈・KPI・短コピーを重ねて訴求装置化
- 抽象的な3D図形を排除
- 競合と差別化される機能を強調

## 採用判断

5軸採点目安（BtoB SaaS 平均）:

```
文脈: 4（商材依存）
共感: 3（業務シーンより画面）
CV: 4（機能訴求強）
差別化: 1〜3（業界全社採用）
実現: 5（制作容易）
合計: 17〜19
```

差別化が弱いため、必ず他パターンと比較推奨。
