# Workflow Diagram Hero

## 概要

業務フロー図解をヒーローに配置するパターン。「こう変わる」を視覚化。

## Best Use Cases

- BtoB 業務改善 SaaS
- ワークフロー自動化系
- 複雑な業務を簡素化する商材
- ペルソナがフローで意思決定

## NG Use Cases

- 業務フローが単純すぎる商材
- 視覚化しても伝わらない抽象サービス

## 構成要素

```
- 左: テキスト（メインキャッチ）
- 右: 業務フロー図
  - Step 1 → Step 2 → Step 3
  - 各ステップに業務内容
  - 改善後の効率を矢印・数字で表現
- 下: メインCTA
```

## CV貢献

- 業務理解（即時）
- 改善期待（具体性）
- 差別化（図解の独自性）

## 例

業務フロー図：

```
[応募者管理]
  Excel → [自社サービス] → ダッシュボード
  月30h     自動連携        月5h（80%削減）

[選考連絡]
  手作業 → [自社サービス] → 自動メール
  月20h     テンプレ           月0h（100%削減）

[経営報告]
  集計   → [自社サービス] → 自動レポート
  月8h     リアルタイム       月0h（100%削減）
```

## 採用判断

```
文脈: 4
共感: 4（業務文脈共感）
CV: 5（具体的改善訴求）
差別化: 5（独自図解）
実現: 3（図解制作工数高）
合計: 21
```
