# Concrete Expression Rules

具体的な表現で書くためのルール。

## 抽象 → 具体 変換の3軸

### 軸1: 数字
- 「効率化」→「月150時間 → 月60時間（60%削減）」
- 「多くの企業が」→「導入企業1,200社（自社調べ・2026年4月）」
- 「すぐに」→「最短2週間」「即日対応」「3分で資料DL」

### 軸2: 場面・時間
- 「業務効率」→「月曜の朝、ダッシュボードを開く」
- 「使いやすい」→「移動中の電車内でも、スマホで応募者対応」
- 「サポート」→「平日9-18時、チャットで即応」

### 軸3: 人物・役割
- 「お客様」→「中小企業の人事担当者」
- 「ユーザー」→「30代エンジニア（3〜5年経験）」
- 「担当者」→「営業部長 田中様（許諾済）」

## 具体化テンプレート

### Before / After 型

```
【Before】
- 業務工数: 月150時間
- 報告書作成: 週8時間
- データ抜け漏れ: 月10件以上

【After（3ヶ月後）】
- 業務工数: 月60時間（60%削減）
- 報告書: 自動生成で実質ゼロ
- 抜け漏れ: ほぼゼロ

※ 自社調べ・100社平均（2026年4月時点）
```

### 1日の流れ型

```
9:00 出社（リモートも可）
9:30 朝会で当日タスク確認
10:00 開発に集中
12:00 ランチ（メンバーと）
13:00 ペアプロ・コードレビュー
15:00 1on1（マネージャーと）
17:00 進捗共有
18:00 退社
```

### 業務シーン型

```
月曜の朝、出社後の最初の業務：

【現状】
1. メールを開く（10分）
2. スプレッドシートを開く（5分）
3. 各求人媒体の応募者を手動コピペ（30分）
4. 上司への報告書作成（30分）

【導入後】
1. ダッシュボードを開く（5秒）
2. 状況確認（5分）
→ 残り時間で、戦略的な業務に集中
```

### 成果数値型

```
A社（製造業 / 100名）

導入前:
- 採用工数: 月150時間
- 内定辞退率: 25%
- 採用単価: 60万円

導入3ヶ月後:
- 採用工数: 月60時間（60%削減）
- 内定辞退率: 17%（8%減）
- 採用単価: 42万円（30%減）

※ A社固有のデータ・許諾済み
```

## 「具体的」な数字の書き方

### NG（曖昧）
```
- 多くの企業が利用
- 高い満足度
- 短期間で導入可能
```

### OK（具体）
```
- 導入企業1,200社（自社調べ・2026年4月時点）
- 顧客満足度95%（[第三者調査会社] 調べ・対象者300名・2025年12月）
- 最短2週間で導入完了（過去30社の平均値）
```

## 「具体的」なシーンの書き方

### NG（業界一般論）
```
業務効率を上げたい
コストを削減したい
品質を向上したい
```

### OK（業務シーン）
```
月曜の朝、応募者リストを見るのが憂鬱
上司から「あの応募者、返信した？」と聞かれてヒヤヒヤ
良い候補者を見落としていないか、夜眠れない
経営層への週次報告に毎週8時間
```

## 「具体的」な人物の描き方

### NG（抽象）
```
お客様
ユーザー
担当者
```

### OK（具体）
```
中小企業（50〜300名）の人事担当者
30〜45歳、人事経験3年以上
採用業務に月100時間以上を費やしている
```

## 注意：仮置き値の扱い

実顧客データがない場合、以下を守る：

```
✅ 「業界平均では〜」（業界データ参照）
✅ 「中小企業向けに設計」（カテゴリ訴求）
✅ 「※自社調べ・想定値」（明示）

❌ 架空の社数を断定（「1,200社導入」と書きながら実際は数十社）
❌ 出典なき数値（景表法リスク）
```
