# Non-Generic Design Rules

AIっぽい・テンプレート的なデザインを排除し、商材文脈に合うデザインにするためのルール。

## AIっぽいデザインの特徴

### NG パターン

#### 1. どのSaaSにも見える青いグラデーションhero
- 紺〜青のグラデーション背景
- 抽象的な3D図形が浮かぶ
- 「DX」「AI」が散りばめられた heroコピー

#### 2. 汎用的な3D人物イラスト
- 同じ人種・性別比のキャラクター
- 仕事してる風のシーン
- パステルカラーの背景

#### 3. 意味の薄い抽象背景
- 抽象未来都市
- 光の粒子
- ニューラルネットワーク風の線
- 商材と関係ない宇宙・自然

#### 4. アイコンだけが並ぶベネフィット欄
- 円形アイコン × 6個
- アイコン + 短い見出しのみ
- 説明が薄い

#### 5. 文字量と余白が整いすぎて印象が薄い
- 完璧すぎるグリッド
- 余白が均等すぎる
- 文字量が一定すぎる

#### 6. CTAがテンプレート的
- 「お問い合わせはこちら」だけ
- 同じ色・同じサイズ・同じ位置
- ベネフィットなし

## 人間らしいデザインの特徴

### OK パターン

#### 1. 商材の利用シーンが分かる
- 実際の管理画面（SaaS）
- 利用シーンの写真（実際の業務環境）
- ユーザーの作業風景

#### 2. ターゲットの業務文脈に合う
- 人事担当者向け：応募者管理画面
- エンジニア向け：コードエディタ
- 経営者向け：ダッシュボード
- 求職者向け：オフィス・社員写真

#### 3. 画像に目的がある
- 装飾ではなく理解促進
- 「この画像で何を伝えたいか」が明確
- 文章では伝わらない情報をビジュアル化

#### 4. 図解が理解を助ける
- ベネフィット → 機能 のフロー図
- Before / After 比較
- 業務フローの可視化
- 組織構造・関係図

#### 5. 証拠・事例・比較が自然に見える
- 数値はバッジ風
- ロゴ列は均等配置
- 比較表はシンプルに
- 事例は写真 + ストーリー

#### 6. CTA周辺に安心材料がある
- CTAの直下に「クレカ不要」「いつでも解約」
- 流れの予告
- 営業電話の有無

## デザイン構成の推奨

### FV
```
PC レイアウト：
[左: テキスト + CTA] [右: ビジュアル]

SP レイアウト：
[テキスト]
[CTA]
[ビジュアル]
```

### 課題提起
- アイコン + チェックリスト形式
- 担当者の声を吹き出しで
- 業務シーンの写真

### 解決策
- 3つのアプローチをカード並び
- 各カードに：アイコン + 見出し + 本文 + スクリーンショット

### 実績
- 数値はバッジ風（大きめ）
- ロゴ列はグレースケール（控えめ）
- お客様の声は顔写真付きカード

### 比較表
- 自社列をハイライト
- ◎ ○ △ × のシンプル評価
- 出典・時期を表下部に小さく

### 価格
- 3プランを横並び
- 推奨プランをハイライト（バッジ）
- 月額 / 年額切替

### FAQ
- アコーディオン形式
- 開閉アイコン明示

### 最終CTA
- 背景色を変えて注目
- CTA を大きく
- 安心要素をバッジ風に

## カラー選定ルール

### NG: 業界標準色をそのまま使用
- BtoB SaaS → 青一色
- 採用 → 緑一色
- EC → ピンク一色

### OK: ブランド独自色 + 業界文脈
- メインカラー（ブランド独自）
- アクセントカラー（CTA用）
- ベースカラー（白 or 薄グレー）
- テキストカラー（黒 or 濃グレー）

コントラスト比 4.5:1 以上必須。

## タイポ選定ルール

### NG
- 派手なゴシック体だけ
- 装飾フォント乱用
- フォントサイズが極端

### OK
- 見出し: 明朝 or ゴシック（読みやすさ優先）
- 本文: 16px 以上、line-height 1.6〜1.8
- 数字: 太字 or 専用フォント

## 余白設計ルール

### NG
- 全セクション同じ余白
- 余白が極端に狭い / 広い
- グリッド完璧すぎる

### OK
- セクション間: 80〜120px (PC) / 48〜64px (SP)
- CTA周辺: 上下 32〜48px の余裕
- 重要要素は余白で目立たせる

## アニメーション

### NG
- 大きなアニメーション（読書を妨げる）
- 自動再生動画（音付き）
- ポップアップ多用

### OK
- スクロール連動の軽いフェードイン
- ホバー時の色変化
- prefers-reduced-motion 対応必須

## チェックリスト

- [ ] FV が「どのSaaSにも見える」になっていない
- [ ] 商材の利用シーンが見える画像
- [ ] 図解で理解促進
- [ ] CTA周辺に安心材料
- [ ] カラー：ブランド独自色 + コントラスト 4.5:1+
- [ ] タイポ：読みやすさ優先
- [ ] 余白：適切なメリハリ
- [ ] アニメーション：prefers-reduced-motion 対応
