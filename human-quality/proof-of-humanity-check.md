# Proof of Humanity Check

LP出力が「人間が現場を理解して作った」ように見えるかを検証するチェックリスト。

## 5つの検証軸

### 1. 競合置き換えテスト

LPコピーの会社名を競合社名に置き換えて、それでも成立するか確認：

```
✅ 不成立 = 自社固有の内容（OK）
❌ 成立 = どの会社にも当てはまる内容（NG）
```

#### 例

```
NG: 「私たちは業界をリードする革新的なサービスを提供します」
→ 競合に置き換えても成立 → AIっぽい

OK: 「[サービス名] は、月150時間の採用業務を月60時間に削減します（自社調べ・100社平均）」
→ 自社固有データ → 人間らしい
```

### 2. 業務シーン再現テスト

ペルソナの「特定の業務瞬間」が描けているか：

```
✅ 「月曜の朝、応募者リストを開くのが憂鬱」
✅ 「上司から『あの応募者、返信した？』と聞かれてヒヤヒヤ」
✅ 「夜眠れない時がある」

❌ 「業務効率を上げたい」
❌ 「コスト削減したい」
❌ 「DXを推進したい」
```

### 3. 担当者の本音テスト

担当者の「本音」が入っているか：

```
✅ 「失敗したくない」
✅ 「上司に怒られたくない」
✅ 「胸を張って報告したい」
✅ 「業務疲弊から解放されたい」

❌ 「効率化したい」
❌ 「成長したい」
❌ 「最適化したい」
```

### 4. 数字の出典テスト

すべての数値に出典があるか：

```
✅ 「導入企業1,200社（自社調べ・2026年4月時点）」
✅ 「平均60%の業務工数削減（自社調べ・100社平均）」
✅ 「[第三者調査会社] 調べ・対象者300名・2025年12月」

❌ 「業界No.1」（根拠なし）
❌ 「最高品質」（根拠なし）
❌ 「多くの企業が」（数字なし）
```

### 5. 商材文脈テスト

画像・デザインが商材文脈に合っているか：

```
✅ BtoB SaaS → プロダクト管理画面
✅ 採用LP → 実在社員の写真
✅ EC → 実際の商品写真

❌ BtoB SaaS → 抽象3Dイラスト
❌ 採用LP → ストックフォトのチーム
❌ EC → 関係ない風景画像
```

## 詳細チェックリスト

### コピー観点

- [ ] 抽象語の連続使用（3つ以上）なし
- [ ] 「最適」「革新」「伴走」「寄り添う」を雑に使っていない
- [ ] 「〜を実現します」を多用していない
- [ ] どの会社にも当てはまるコピーがない
- [ ] 業務の特定瞬間が描けている
- [ ] 担当者の本音が入っている
- [ ] 「みなさん」「すべての方」だけで終わっていない
- [ ] 数字に出典付き
- [ ] 機能羅列で終わっていない
- [ ] ベネフィットを業務 / 心理 / 評価 / 未来 で分解

### デザイン観点

- [ ] FVが「どのSaaSにも見える」になっていない
- [ ] 商材の利用シーンが見えるビジュアル
- [ ] 抽象的な装飾画像が過剰でない
- [ ] AI生成画像の不自然さがない
- [ ] アイコン・イラストが汎用素材だけになっていない
- [ ] 図解で理解促進
- [ ] CTA周辺に安心材料

### 構造観点

- [ ] LP Structure Blueprint の必須項目を網羅
- [ ] セクション間の流れが自然
- [ ] 課題提起 → 解決策 → 実績 → 比較 → FAQ → CTA の論理
- [ ] FAQ が最終CTA直前

### CV直前観点

- [ ] CTA文言が「動詞 + ベネフィット」
- [ ] 「お問い合わせ」だけになっていない
- [ ] 安心要素（解約・返金・営業電話の有無）
- [ ] CV後の流れ予告

## 検証スコア

各項目1点として、人間らしさスコアを算出：

| スコア | 判定 |
|---|---|
| 25-30点 | 高品質（人間らしい） |
| 20-24点 | 中品質（軽微修正） |
| 15-19点 | 低品質（主要改善必須） |
| 14点以下 | 作り直し |

## 検証フロー

1. LP全体を通読
2. 上記チェックリストで採点
3. 不合格項目を特定
4. 該当 skill / agent で修正
5. 再採点
