# Fogg 行動モデル

## 公式
B = MAP

- **B**ehavior（行動）= **M**otivation（動機）× **A**bility（能力）× **P**rompt（きっかけ）

## LPでの使い方

### Motivation（動機を高める）
- ベネフィット強調
- 限定性 / 緊急性
- 社会的証明（他者がやっている）
- 将来の理想像

### Ability（できる能力にする）
- フォーム項目最小化
- ステップを少なく
- マイクロコピーで迷わせない
- スマホ最適化

### Prompt（きっかけを作る）
- CTA の明示
- 追従CTA
- 離脱ポップアップ
- 限定性表示（残り○名）

## 「行動が起きない」ときの診断
- 動機が弱い → ベネフィット強化 / 不安解消
- 能力が低い → フォーム削減 / UI改善
- きっかけがない → CTA配置追加

## 注意点
- 偽の限定性（常時セール表記）は景表法違反
- 不安煽りで動機を作るのは長期的にブランド毀損
