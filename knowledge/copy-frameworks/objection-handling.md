# 反論処理（Objection Handling）

## 主要な反論カテゴリ

| カテゴリ | 例 | 対応 |
|---|---|---|
| 価格 | 高い・予算が合わない | ROI / 分割 / 無料トライアル |
| 効果 | 本当に効くのか | 実績 / 事例 / 保証 |
| 時間 | 導入に時間がかかる | 最短期間 / サポート手厚さ |
| 既存代替 | 既に他社使っている | 比較・差別化・移行支援 |
| 自社事情 | うちには合わない | 業種別事例 |
| 解約 | 縛りが心配 | 解約条件明示 |
| 信頼 | この会社大丈夫か | 会社情報 / 受賞 / 認証 |

## 反論処理の3ステップ

1. **共感**: 「ご不安ですよね」
2. **論拠**: 「実は[根拠] があります」
3. **証拠**: 「実際に[事例] です」

## LPでの配置箇所

- 反論を直接書くセクション: FAQ
- 暗黙の反論への対応:
  - 価格 → 価格セクションで分割払い・ROI試算
  - 効果 → 実績セクションで数値・事例
  - 信頼 → 会社情報・受賞・認証セクション

## 反論リストの作り方

1. ペルソナ・営業ヒアリング・カスタマーサポートから収集
2. 10〜20件リストアップ
3. 主要5〜10件をFAQ化
4. それ以外は暗黙的に各セクションで対応
