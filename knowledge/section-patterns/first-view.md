# ファーストビュー（FV）パターン

## FV の役割
- 3秒で「誰の」「何が得られる」を伝える
- 信頼バッジで離脱を防ぐ
- 主CTAを目立たせる

## 必須要素
- メインキャッチ
- サブコピー
- メインCTA
- 信頼バッジ（実績数値・認証・受賞 等）
- ビジュアル（プロダクト or 利用シーン）

## 高CVRの型

### 型A: ベネフィット型
```
[誰の][課題] を [改善幅] する [カテゴリ名]

[サブコピー：付加情報]

[CTA: 行動]   ★[信頼バッジ]
```

### 型B: 数字型
```
[業界 / カテゴリ] No.1 の [サービス名]
[XX,XXX] 社が導入

[CTA]
```

### 型C: 質問型
```
[ターゲット] の方へ
[共通の悩み] でお困りではないですか？

[CTA]
```

## 悪い例
- 商品名だけ「○○Cloud」
- 抽象的「世界を変える」
- ベネフィットなし「最高の品質」
- CTAなし
- 信頼バッジなし

## BtoB FV
- 課題＋解決メッセージ
- 導入企業ロゴ列
- 「無料で資料請求」CTA + 「無料でデモを見る」サブCTA

## 採用LP FV
- ミッション or 仕事の魅力
- 社員の写真
- 「カジュアル面談を予約」CTA

## CTA設計
- 行動動詞 + ベネフィット
- 例: 「無料で資料を受け取る」「3分で診断する」「今すぐ予約する」

## チェックリスト
- [ ] メインキャッチが3秒で意味取れる
- [ ] サブコピーが補足機能
- [ ] CTAが目立つ（コントラスト・サイズ）
- [ ] 信頼バッジあり
- [ ] SP表示で破綻しない
