# 問題提起セクション

## 役割
- ターゲットの悩みを言語化
- 「自分のことだ」と思わせる
- 解決策セクションへの橋渡し

## 構成パターン
- 見出し: 「こんな悩みありませんか？」
- 悩みリスト 3〜5個（チェックリスト風）
- 共感文

## 悩みリストの書き方
- 抽象的なものより具体的なもの
- 業務シーンが浮かぶ表現
- 数字を入れると刺さる（「月X時間」「年Y万円」）

## 例（BtoB SaaS）
```
こんな課題、ありませんか？

□ 採用業務に月100時間以上を費やしている
□ 応募者管理がスプレッドシートでバラバラ
□ 良い候補者を見落としてしまうことがある
□ 上長への報告に時間がかかる
□ 採用効果の可視化ができていない

ひとつでも当てはまったら、続きをお読みください。
```

## 注意点
- 不安煽り過剰NG
- 解決できない大きな課題を提起しない（嘘になる）
- 法的に問題のある効果効能訴求NG
