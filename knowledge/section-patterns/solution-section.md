# 解決策セクション

## 役割
- 自社サービスでどう解決するかを示す
- 機能ではなくベネフィットで訴求
- 強みセクションへの橋渡し

## 構成パターン
- 見出し: 「[サービス名] が解決します」
- リード文: 解決アプローチの全体像
- 解決アプローチ 3個（アイコン + 見出し + 本文）

## ベネフィット翻訳の基本
- 機能: 「APIで他システムと連携」
- ベネフィット: 「既存ツールとの二重入力をなくし、月X時間削減」

## 例（BtoB SaaS）
```
[プロダクト名] が解決します

採用業務を一元管理し、求職者対応を自動化することで、
採用工数を最大50%削減します。

1. 応募者を一元管理
   バラバラな応募経路を一画面で管理。スプレッドシート卒業。

2. 候補者へのメッセージを自動化
   選考結果の自動返信で、対応漏れゼロ。

3. ダッシュボードで効果可視化
   経路別CV率・採用単価がリアルタイムで分かる。
```

## 注意点
- 機能羅列に陥らない
- 効果の根拠（数値）は出典明記
- 法的に問題のある効果訴求は避ける
