# First View Requirements

LP最重要パーツである FV の必須条件。

## 必須6条件

```
1. 3秒で誰向けか分かる
2. 3秒で得られるメリットが分かる
3. 信頼できる理由が最低1つある
4. CTAが見える
5. 画像・ビジュアルが訴求と一致している
6. AIっぽい汎用heroになっていない
```

## 各条件の詳細

### 1. 3秒で誰向けか分かる

**良い例**:
- 「中小企業の人事担当者へ」（明示）
- 「BtoB SaaS の経営者向け」
- 「スタートアップで働く30代エンジニアの方へ」

**悪い例**:
- 「すべての方に」（広範囲すぎ）
- 「みなさまへ」（誰向けか不明）
- ターゲット明示なし

### 2. 3秒で得られるメリットが分かる

**良い例**:
- 「採用業務を最大60%削減」（数値ベネフィット）
- 「契約書チェック工数を3分の1に」
- 「最短2週間で導入完了」

**悪い例**:
- 「業界最高のサービス」（抽象）
- 「革新的なソリューション」（意味不明）
- 商品名だけ「TalentFlow」

### 3. 信頼できる理由が最低1つ

**良い例**:
- 「導入企業 1,200社（自社調べ・2026年4月時点）」
- 「ISO27001 / Pマーク取得済み」
- 「[第三者調査] 顧客満足度No.1（YYYY年）」

**悪い例**:
- 「業界No.1」（根拠なし）
- 信頼バッジなし
- 「最高品質」（抽象）

### 4. CTAが見える

**良い例**:
- メインCTA + 副CTA（電話 等）
- コントラスト 4.5:1 以上
- ボタン文言が「動詞 + ベネフィット」

**悪い例**:
- CTA未配置
- スクロールしないと見えない
- 背景に埋もれる

### 5. 画像・ビジュアルが訴求と一致

**良い例**:
- プロダクト画面（SaaS）
- 利用シーン（実在感）
- ターゲット業務文脈に合う図解

**悪い例**:
- 関係ないストックフォト
- 抽象的なグラデーション背景のみ
- AI生成感のある不自然な3Dイラスト

### 6. AIっぽい汎用heroになっていない

**NG パターン**:
- どのSaaSにも見える青いグラデーションhero
- 汎用的な3D人物イラスト
- 意味の薄い抽象未来都市
- 商材と関係ないテンプレート画像

**OK パターン**:
- 商材の利用シーンが分かる
- ターゲットの業務文脈に合う
- 図解・実画面・現場写真を優先

## FV構造の型

### 型A: ベネフィット型（BtoB SaaS推奨）

```
[ターゲット明示]

[メインキャッチ: 業務 + 改善幅]

[サブコピー: 補足説明]

[メインCTA] [副CTA]

★[信頼バッジ] ★[認証] ★[実績数値]

[プロダクト画面]
```

### 型B: 課題訴求型（コンサル / 高単価推奨）

```
[ターゲット明示]

[共通の悩みを言語化したキャッチ]

[サブコピー: 解決アプローチ]

[メインCTA: 無料相談を予約]

★[権威性: 経歴・著書・実績]
```

### 型C: ストーリー型（採用LP推奨）

```
[ミッション or 仕事の魅力]

[サブコピー: 価値・特徴]

[メインCTA: カジュアル面談]

★社員数 / 受賞 / メディア掲載

[実在社員の写真（許諾済）]
```

## チェックリスト

- [ ] 3秒テスト合格
- [ ] ターゲット明示
- [ ] ベネフィット具体（数値・事例）
- [ ] 信頼バッジ最低1つ
- [ ] CTAコントラスト 4.5:1+
- [ ] 商材文脈に合うビジュアル
- [ ] AIっぽい汎用 hero でない
- [ ] SP表示で破綻なし
