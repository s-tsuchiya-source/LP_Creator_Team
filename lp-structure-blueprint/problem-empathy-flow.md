# Problem Empathy Flow

課題提起 → 共感 → 放置リスク の流れを設計する。

## 役割

- ユーザーが「自分のことだ」と感じる
- 「分かってくれている」と信頼を得る
- 課題の重要性を再認識させる
- 解決策セクションへの橋渡し

## 必須要素

- ターゲット明示
- 言語化されていない悩みの言語化（3〜5個）
- 共感文
- 放置リスクの提示（暗示）

## 良い例

### 型A: 「こんな悩みありませんか」型

```markdown
## こんな悩み、ありませんか？

□ 月曜の朝、応募者一覧のスプレッドシートを開くのが憂鬱
□ 「あの応募者、返信した？」と上司に聞かれるたびにヒヤヒヤ
□ 良い候補者を見落としていないか、夜眠れない時がある
□ 経営層への報告に毎週8時間も使っている
□ 採用効果がデータで見えていない

ひとつでも当てはまったら、続きをお読みください。
```

### 型B: 担当者の本音型

```markdown
## 「なんでうちの採用、うまくいかないんだろう」

応募者対応に追われ、本来やるべき面接準備が後回しに。
気づけば、せっかくの優秀候補者を逃してしまう。

そんな経験、ありませんか？
```

### 型C: 放置リスク型

```markdown
## このまま、業務が回らなくなる前に

採用業務に追われ続けて1年。
担当者は疲弊し、応募者対応の遅延は週10件以上。

放置すると：
- 内定辞退率の上昇
- 担当者のバーンアウト
- 経営層からの不信
- 採用予算の削減

そして来年、また同じ繰り返し。
```

## 悪い例

### NG型1: 課題を箇条書きで並べるだけ

```
- 効率化したい
- コスト削減したい
- 品質向上したい
```

→ 抽象すぎ。誰のことか分からない。

### NG型2: どの業界にも当てはまる表現

```
- 業務効率を上げたい
- DX を進めたい
- 競合に勝ちたい
```

→ 業界特性を捉えていない。

### NG型3: 不安を煽るだけ

```
このまま放置すれば会社は終わります。
今すぐ行動しないと手遅れになります。
```

→ 不安商法。信頼毀損。

### NG型4: 共感文がテンプレ

```
お困りでしょう。
私たちが解決します。
```

→ 当事者意識ゼロ。

## 担当者の本音を入れるコツ

- 「上司に怒られたくない」
- 「失敗したくない」
- 「夜眠れない時がある」
- 「胸を張って報告したい」
- 「業務疲弊から解放されたい」

抽象論ではなく、**業務の特定瞬間**で表現する。

例：
- ❌「効率化したい」
- ✅「月曜の朝、スプレッドシートを開くのが憂鬱」

## 放置リスクの見せ方

### 良い例（事実ベース）

```
業務工数が増え続けると、来期の採用予算が削減される可能性があります。
担当者のバーンアウトは、組織全体の採用力低下に繋がります。
```

### 悪い例（不安煽り）

```
このまま放置すると、会社は危機に陥ります。
来年には倒産しているかもしれません。
```

## Validation 観点

- [ ] ターゲット明示
- [ ] 言語化されていない悩み 3〜5個
- [ ] 業界一般論ではなく、特定業務シーンで表現
- [ ] 担当者の本音が入っている
- [ ] 不安煽り過剰でない
- [ ] 解決策セクションへの橋渡しがある
