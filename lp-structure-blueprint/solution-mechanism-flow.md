# Solution Mechanism Flow

解決策と「なぜそれで解決できるのか」のメカニズムを設計する。

## 役割

- このLPが提示する解決の核心メッセージ
- なぜ解決できるのかを論理的に説明
- ベネフィット → 機能 の順序で説明
- 「これなら解決できそう」と確信させる

## 必須要素

- 解決約束の核心メッセージ
- 解決メカニズム（3〜5個のアプローチ）
- 機能とベネフィットの紐付け
- スクリーンショット / 図解

## 良い例

### 型A: 解決約束 + 3アプローチ型

```markdown
## [サービス名] が解決します

採用業務を一元管理し、求職者対応を自動化することで、
採用工数を最大60%削減します（自社調べ・100社平均）。

### 1. 応募者を一元管理
**ベネフィット**: スプレッドシート卒業、抜け漏れゼロ
**機能**: API連携で求人媒体から自動取込

### 2. 候補者へのメッセージを自動化
**ベネフィット**: 対応漏れゼロ、応募者の体験向上
**機能**: テンプレートメール + ステータス連動

### 3. ダッシュボードで効果可視化
**ベネフィット**: 経営層への報告が自動、戦略判断が早まる
**機能**: リアルタイムレポート + カスタマイズ可能
```

### 型B: Before / After 型

```markdown
## 導入前 → 導入後

### 導入前（現状）
- 採用業務 月150時間
- 応募者対応の遅延 30件/月
- 経営層への報告が手作業

### 導入後（3ヶ月後）
- 採用業務 月60時間（60%削減）
- 応募者対応の遅延 0件/月
- ダッシュボードで自動レポート

※ 自社調べ・100社平均
※ 効果には個人差・業種差があります
```

### 型C: メカニズム明示型

```markdown
## なぜ採用工数を 60% 削減できるのか

[サービス名] は、採用業務の3つの「無駄」を排除します：

### 1. 二重入力の無駄
求人媒体・応募者管理・メール送信が別ツールで動いている。

→ API連携で自動連動。1度入力すれば全システムに反映。

### 2. 対応漏れの無駄
「あの人、返信した？」を気にする時間が業務時間を圧迫。

→ ステータス管理 + 自動メールで対応漏れゼロ。

### 3. 報告書作成の無駄
週次・月次の報告書を手作業で作成。

→ ダッシュボードからワンクリックで自動生成。

これら3つの無駄を排除することで、平均60%の工数削減を実現します。
```

## 悪い例

### NG型1: 機能羅列のみ

```markdown
## 主要機能

- API連携
- 自動メール
- ダッシュボード
- アクセス権限管理
- データエクスポート
```

→ 「だから何？」となり離脱。

### NG型2: 抽象的な解決約束

```markdown
## 採用業務を効率化します

私たちのサービスは、あなたの採用業務を変えます。
最高の機能で業界を革新します。
```

→ 何も伝わらない。

### NG型3: 機能 → ベネフィットの順序

```markdown
### 1. API連携機能（機能名先行）
当社のAPIは...

→ 機能の説明が長く、ベネフィットが見えない。
```

## ベネフィット → 機能 の順序

7段階変換（`skills/08_experience/benefit-translation-skill.md` 参照）：

```
[1. 機能]
   ↓
[2. 何が楽になるか]
   ↓
[3. 何が減るか]
   ↓
[4. 何が増えるか]
   ↓
[5. 誰に評価されるか]
   ↓
[6. どんな未来が得られるか]
   ↓
[7. CVする理由]
```

LP上での表現順序は逆：

```
[ベネフィット（業務 / 心理 / 評価 / 未来）] → [機能の裏付け]
```

## メカニズム説明のコツ

### 「なぜ」を3層で説明

```
ベネフィット: 採用工数 60%削減

なぜ削減できる？
↓
3つの無駄を排除（二重入力 / 対応漏れ / 報告書）

なぜ排除できる？
↓
API連携 / 自動メール / ダッシュボード

なぜ動く？
↓
業界標準のWebhook規格 + 主要HRシステムとのパートナーシップ
```

LPでは2層目までが基本。3層目は無料相談・資料請求で。

## Validation 観点

- [ ] 解決約束の核心メッセージあり
- [ ] 3〜5個のアプローチ
- [ ] ベネフィット → 機能 の順
- [ ] 数値根拠（出典付き）
- [ ] スクリーンショット / 図解あり
- [ ] 「個人差」明記
