# output-formats.md — 出力フォーマット規定

各フェーズで生成する成果物のフォーマット規定。

## 1. Brief（ブリーフ）

```markdown
# LP制作ブリーフ: [案件名]

## 基本情報
- 案件名:
- 作成日:
- ディレクター:
- 想定公開時期:

## 商材
- サービス名:
- カテゴリ:
- 価格帯:

## 目的・KPI
- LP目的:
- 主要CV:
- 補助CV:
- KPI数値目標:

## ターゲット
- 主ターゲット:
- 副ターゲット:
- 業種 / 職種 / 年齢:

## 提供価値
- 主ベネフィット:
- 副ベネフィット:

## 競合・差別化
- 競合:
- 差別化要素:

## 実績・証拠
- 利用社数 / 導入事例:
- 数値実績:
- 受賞・認証:

## 制約
- 公開期日:
- 法的制約:
- ブランドガイドライン:

## 出力形式
- 戦略のみ / コピー込み / 実装込み / 計測込み:

## 仮置き項目（要確認）
- ...

## 確認が必要な項目
- ...
```

## 6. Code（実装）

ディレクトリ構成:

```
outputs/06_code/[案件名]/
├─ index.html
├─ style.css
├─ script.js
└─ README.md  ← 実装メモ
```

実装メモには以下を含める:

- 想定動作環境
- ビルド手順
- 計測用 data 属性
- フォーム送信先
- 注意点

## 8. QA Checklist（QAチェックリスト）

```markdown
# QAチェックリスト: [案件名]

## 表示
- [ ] 主要ブラウザ表示確認
- [ ] レスポンシブ確認

## リンク
- [ ] 全リンク到達確認
- [ ] 外部リンク target=_blank

## フォーム
- [ ] 必須バリデーション
- [ ] 送信成功確認
- [ ] エラーメッセージ

## 計測
- [ ] GA4 イベント発火
- [ ] CV設定

## 法務
- [ ] 誇大表現なし
- [ ] No.1表現の根拠
- [ ] 薬機法 / 景表法

## 速度
- [ ] LCP < 2.5s
- [ ] CLS < 0.1
- [ ] INP < 200ms

## a11y
- [ ] alt 設定
- [ ] フォーカスリング
- [ ] キーボード操作
```
