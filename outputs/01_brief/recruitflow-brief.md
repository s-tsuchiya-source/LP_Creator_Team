# LP制作ブリーフ — RecruitFlow（採用管理クラウド）

## 案件概要

- **案件名**: RecruitFlow 資料請求LP
- **業種分類**: BtoB SaaS（HR Tech / 採用管理）
- **採用 industry-playbook**: `industry-playbooks/btob-saas-playbook.md`
- **LPタイプ**: 新規LP × BtoB高単価寄り × 計測あり

## 目的とKPI

- **主CV**: 資料請求フォーム送信
- **副CV**: なし（最初は主CV1本に集中させる）
- **想定流入**: 検索広告 / 比較記事 / 直接 / メール
- **KPI**: フォーム送信率（CTR×CVR）、フォーム到達率、SP/PC別CVR

## ターゲット（仮置き）

- **担当者**: 人事・採用担当（応募対応に追われる現場）
- **上長**: 人事マネージャー・採用責任者
- **決裁者**: 人事部長 / 経営層（中堅企業のため近い）
- **情シス**: セキュリティ確認のため資料を見る

## 中核訴求軸（仮説）

1. 媒体・応募・選考を一画面に集約 → 入力転記をなくす
2. LINE / メール / 電話メモを履歴として残す → 属人化を抑える
3. マネージャー権限分離 → 現場が触れる範囲を絞る → 情シス安心

## 法務・表現の方針

- 「業界No.1」「最速」「100%」など断定表現は使わない
- 数値実績は出典なしのため掲載しない（FV含め）
- 認証バッジ（ISO27001 / Pマーク等）も実態未確認のため記載しない
- ベネフィットは「〜できます」「〜になります」までに留める

## 出力範囲

- ブリーフ（本ファイル）
- 戦略 / 構成 / コピー / デザイン方針
- HTML / CSS / JS（動作する形）
- GA4 / GTM 計測設計
- QAチェックリスト
- 各 validation log
- final-acceptance-report

## 仮置き宣言

本案件は架空のテスト案件のため、以下を仮置きとして進行する：
- ターゲットの規模感
- 競合差別化の細部
- 実績数値（**載せない**ことで安全側に倒す）
- 連携媒体の具体名

実案件運用時はユーザー再ヒアリング必須。
