# 戦略設計 — RecruitFlow 資料請求LP

## CV戦略

- **主CV**: 資料請求（PDF送付）
- **CV後**: 翌営業日までに営業から日程調整連絡 / 営業電話なしを選べる
- **副CV**: なし（フォーム1本に集中）

## ターゲット階層と訴求の使い分け

| 役割 | 主な関心 | 主な訴求 |
|---|---|---|
| 採用担当（ユーザー） | 業務工数 | 入力転記が消える / 媒体横断確認 / LINE通知 |
| 採用責任者 | チーム全体の見える化 | 進捗ダッシュボード / 担当者依存からの脱却 |
| 人事部長・経営層 | 採用コスト・歩留まり | 媒体ごとの応募〜内定率を可視化 |
| 情シス | セキュリティ・運用 | 権限分離 / 監査ログ / SSO対応 |

## 訴求軸（3本）

1. **応募〜内定までを一画面で運用**（情報サイロ解消）
2. **応募者の状況を全員で共有**（属人化の解消）
3. **現場マネージャーに必要な範囲だけ見せる**（情シス安心）

## Emotional CVR Layer 反映方針

- **メリット**: 機能ではなく「月末の媒体集計が消える」「電話メモを探さなくていい」など現場文脈に翻訳
- **期待**: 大胆な数値ではなく「導入企業の声」「30分のオンラインデモ」など根拠ある期待
- **不安解消**: 価格・導入期間・移行手間・解約・社内説明・効果の6カテゴリでFAQに対応
- **自分ごと化**: 「中堅企業 / 採用担当 / 媒体5つ以上」など具体プロフィールでターゲット明示
- **CV後の安心**: 「営業電話なしを選べる」「資料はPDF / 24時間以内」を明記

## 競合代替に対するポジショニング

- vs Excel運用 → 進捗が止まらない / 引き継ぎが楽
- vs 既存ATS → 媒体連携の広さ / 現場権限の細かさ
- vs 紙台帳 → 検索性 / 履歴 / 監査
