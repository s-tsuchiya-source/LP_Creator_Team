# Photo Annotation Hero Plan - LP3 (運用代行)

## 案件
- 対象: LP3 (運用代行 / lp3-operation-outsourcing)
- 適用日: 2026-04-29

## 1. 写真選定
- **採用写真**: 採用担当者がデスクで業務している様子
- **業務文脈**: 採用業務の現場（書類・PC・電話対応）
- **採用理由**: ペルソナ（採用担当者）の業務シーン即時伝達

### 不採用候補
- 抽象オフィス風景: 業務文脈薄い
- ストックフォトの笑顔チーム: 実在感ゼロ
- 関係ない自然・都市風景: 商材無関係

## 2. オーバーレイ
- **種別**: ブランドカラー（紺）グラデーション
- **透過率 PC**: 60〜45%（写真が透ける）
- **透過率 SP**: 75〜60%（文字視認性優先）
- **視認性**: コントラスト 4.5:1 以上確認

```css
.lp3-hero-photo-overlay {
  background: linear-gradient(
    135deg,
    rgba(13, 71, 161, 0.65) 0%,
    rgba(13, 71, 161, 0.45) 100%
  );
}

@media (max-width: 640px) {
  .lp3-hero-photo-overlay {
    background: linear-gradient(
      135deg,
      rgba(13, 71, 161, 0.80) 0%,
      rgba(13, 71, 161, 0.65) 100%
    );
  }
}
```

## 3. メインコピー
- **メインキャッチ**: 「採用の運用、引き受けます」（13文字）
- **サブコピー**: 「中小企業の採用業務に専念できる体制」（19文字）

## 4. 注釈
- **配置**: 写真右上（業務量を指す）
- **内容**: 「業務時間 月150h → 月60h」 + 注記「※ 例・自社調べ想定値（本番では実数値に差し替え）」
- **指線**: 細い白線（border 1px solid rgba(255,255,255,0.6)）
- **数値の安全性**: HTML上で `<small>※ 例・自社調べ想定値（本番では実数値に差し替え）</small>` を付帯。景表法の優良誤認回避のため、本番公開前に営業/CS で実数値・出典・許諾を確認すること（裏付けが取れない場合は削除またはシミュレーション表記へ置換）。

## 5. KPI バッジ
- **配置**: 写真左下（コーナーバッジ）
- **内容**: 「導入1,200<small>社</small>」 + 注記「※ 例・想定値」
- **内容2**: 「専任担当が伴走」（期間表記は実数値が確定するまで省略）
- **数値の安全性**: HTML上で `<em>※ 例・想定値</em>` を併記。確定実数値が入ったタイミングで注記を取り外す運用。

## 6. CTAボタン
- **文言**: 「3分で資料を受け取る」（11文字）
- **副CTA**: 「無料相談を予約」（8文字）

## 7. レスポンシブ
- **PC**: 写真背景全面（ブランドカラー オーバーレイ）+ テキスト中央寄せ
- **SP**: 写真縦長クロップ + テキスト中央集中
- **ブレークポイント**: 640px

## 8. CV貢献マップ

ペルソナ視線誘導順:
1. ターゲット明示「中小企業の採用担当者へ」（自分ごと化）
2. メインキャッチ「採用の運用、引き受けます」（メリット）
3. KPIバッジ「導入1,200社」（信頼）
4. 注釈「月150h → 月60h」（期待）
5. CTAボタン（行動）

## 採用理由ログ

### 写真採用理由
- 運用代行は「人が業務を引き継ぐ」感情訴求が重要
- 業務シーン写真でペルソナ即時共感
- 競合SaaSの画面UI訴求から差別化

### 注釈採用理由
- 単なる写真背景は装飾的
- 注釈で「具体的な改善数値」を可視化
- 写真の特定箇所（PC作業量）を指して意味付け

### KPI バッジ採用理由
- 数字で実在感
- 「専任担当」で運用代行の特長訴求
- コーナー配置で視線誘導

## 実装状況（本フェーズ反映）

- LP3 `index.html` のヒーローを `lp3-hero--photo` クラスに置換し、`assets/images/lp3/img_hero_01.png` を `<img class="lp3-hero__bg">` で配置
- 既存のタスクボードはヒーロー直下の `lp3-board-section`（「引き取る業務の見える化」）に移設し、ヒーローの責務を「シーン共感 + 期待生成」に純化
- KPI / 注釈 / CTA ブロックは `data-reveal data-motion="fade|scale"` で控えめに reveal、CV直前モーション禁止ルール準拠
- PC: オーバーレイ 60〜45% / SP: 75〜60% を `lp3.css` に実装、`prefers-reduced-motion: reduce` で reveal 無効化済み
- メインキャッチ「媒体運用を、引き受けます。」は `<br class="pc">` で PC のみ改行、SP は一行表示

## 最終判定
- photo-annotation-hero-scorecard: 88/100
- 合格基準（85+）クリア
- 残課題: 数値の本番差し替え運用（営業/CS 確定値入手後に注記を外す）
