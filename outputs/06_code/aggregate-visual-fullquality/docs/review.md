## ビジュアル面レビュー

### 「画像 = 訴求装置」要件の検証

各LPで以下が成立している：

- **写真にグラデの編み**: `.photo-billboard__frame::before` で linear-gradient + radial-gradient 編み実装
- **写真上に短コピー**: `.photo-billboard__title` を写真の上に配置
- **写真上にKPIカード**: `.billboard-card` 4枚を写真右側にレイアウト
- **UI断片やグラフのオーバーレイ**: hero SVG に複合チャート・KPIカード・リング図を統合
- **図解として整理**: ファネル図 / PDCA / 予算配分 / プロセスフロー の独立SVG
- **画像とイラストの組み合わせ**: 実写PNG + SVGビジュアル + HTMLオーバーレイ の3層

| 観点 | 結果 |

| ストックフォト感 | 🟡 写真は自動生成された素材だが、グラデ編み + KPI + コピー重ね合わせで「訴求装置化」している |

## UI/UX面レビュー

### 文字サイズ・余白
- 見出し: clamp() で 28px〜60px の自動スケール
- 本文: 14.5〜17px、line-height 1.85〜2.0 で読みやすさ確保
- セクション余白: clamp(72px, 9vw, 120px) で画面サイズに追従

### 視認性
- 配色: 全てWCAG AA準拠を意識（暗い背景には #fff or 高彩度アクセント）
- フォーカスリング: `:focus-visible` で 2px outline + 3px offset
- タッチターゲット: btn は最小 44px × 44px 確保

### 情報の流れ
- 各LPで「FV → ペイン → 解決 → フロー → 信頼 → FAQ → CTA」の流れを保持
- セクション間に photo-billboard を挟んで「動機の維持」を実現

### スマホ対応
- 760px 以下で全コンテンツ 1カラム化
- sticky-cta は SP のみ表示（trigger 越え時にスライドイン）
- HEROビジュアルは aspect-ratio を SP 用に調整

### CTA挙動
- 主CTA: 全て `.btn--primary` （LPカラーグラデ）
- 副CTA: `.btn--ghost` （アウトライン）
- スマホ底部: sticky-cta で常時2ボタン
- スムーススクロール + ヘッダーオフセット計算
