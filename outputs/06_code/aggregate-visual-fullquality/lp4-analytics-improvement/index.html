EDITORIAL / 採用データの現実

「見ている数字」と「効く数字」は、違う。

採用データの取り扱いに、共通する4つのつまずきがあります。

01 — REPORT

レポートは届くが、
「何を意思決定すべきか」が書かれていない。

毎月送られてくるレポートには、応募数・CPA・採用数が並ぶ。けれど、来月の打ち手は載っていない。担当者の頭の中だけに、次のアクションが残されている。

02 — TOOL

媒体ごとの管理画面を
行き来するだけで、1日が終わる。

媒体A、媒体B、媒体C。それぞれ違うUIで、違う数字の出し方。横串で比較する仕組みがないから、判断材料は毎月Excelに転記される。

03 — INSIGHT

分析は属人化していて、
担当者が変わると一からやり直し。

数字を読む力は、現場の暗黙知になりがちだ。「この数字が動いたら、ここを見る」というナレッジが、ドキュメントとして残っていない。

04 — ACTION

分析と改善のあいだに、
大きな空白がある。

レポートは出る。原稿差替も行う。けれど「分析の結果が、本当に改善につながったか」を後追いで確認する仕組みは、ほとんどの組織にない。

PDCA × DATA

データを起点にした、採用改善サイクル。

「見る」だけのデータから、「動かす」ためのデータへ。aggregateの分析支援は、4ステップで設計されています。

P

計画 PLAN

過去データから「下げる/上げる指標」を特定。月次目標を媒体・原稿・職種ごとに設定し、改善仮説を文章で残します。

D

実行 DO

仮説に基づいた配分変更・原稿差替・CTA改善を実施。「いつ、何を、なぜ変えたか」をログとして記録します。

C

検証 CHECK

変更前後の指標を媒体・原稿・職種・期間で比較。仮説が当たったか / 外れたかを、数字とコメント両方で残します。

A

改善 ACT

当たった仮説は標準化、外れた仮説は原因分析。学びをナレッジ化し、次の月のPLANにつなげます。

指標一覧

「見ると、動かせる」主要指標。

aggregateが標準で追跡する38指標のうち、月次レビューで頻出する代表7指標を抜粋しました。

M-001 / VOLUME
媒体別 応募数
どの媒体が母集団を生んでいるかを把握。応募数だけでなく「採用までの歩留まり」と組合せて評価します。
M-002 / COST
媒体別 応募単価（CPA）
主要KPI。職種・期間別にも分解し、停止候補・増額候補の判断材料にします。
M-003 / FUNNEL
閲覧→詳細→応募 の歩留まり
原稿の魅力訴求が効いているか、応募導線に摩擦があるかを段階別に可視化します。
M-004 / HIRE
職種別 採用率（応募→採用）
応募数だけでなく「採用に至る応募」の割合を測定。媒体・原稿の質の評価軸になります。
M-005 / TIME
職種 × 週 ヒートマップ
どの職種を、いつ出稿するかの根拠を作るためのデータ。閑散期の無駄出稿を減らします。
M-006 / TREND
3ヶ月トレンド（MoM/YoY）
単月では見えない構造変化を捕捉。経営層への報告にも使える長期視点を提供します。
M-007 / IMPACT
改善前後の差分（Before/After）
原稿差替・配分変更の効果を、施策ごとに定量化。「改善が効いた / 効かなかった」を後追いで評価できます。
DASHBOARD / 月次レビュー
REVIEW

「読む」から、「動かす」へ。

レポートは届いて終わり、ではありません。aggregateの月次レビューでは、数字の読み方と次の打ち手を、その場で会話して固めていきます。

よくある質問

相談前に、気になること。

既存のレポートツールやBIを使っていますが、置き換えになりますか？
置き換えではなく、「採用に特化した視点」を補強する位置づけです。既存のBI/レポート基盤と並走させ、採用意思決定に必要な指標だけを抽出してお渡しします。
データの連携にエンジニアリングは必要ですか？
基本はCSVエクスポート / 共有スプレッドシート連携で十分です。本格的なAPI連携を希望される場合は別途ご相談しています。
分析だけでなく、改善実行までサポートできますか？
はい。分析→提案→改善実行→効果検証までを一連でご支援するスコープも用意しています。「分析だけ」「実行だけ」も切り出し可能です。
分析結果を経営層に説明する資料も作ってもらえますか？
A4 1枚の経営層向けサマリーと、現場用の詳細台帳を分けてお渡しします。報告会への同席も対応可能です。
取得しているデータが少ないのですが、それでも分析できますか？
手元の数字でできる範囲を整理し、不足している指標は「次の月から取り始める」設計をご提案します。最初から全指標は揃わなくて構いません。
