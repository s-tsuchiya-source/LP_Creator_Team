# Non Dashboard Hero Scorecard

## Purpose

ダッシュボード型ヒーローのデフォルト化を防ぎ、商材文脈に合うヒーロー選定を採点する（100点満点）。

## Score Breakdown

| 評価項目 | 配点 | 判断基準 |
|---|---:|---|
| 1. ダッシュボード型回避 or 強い根拠 | 30 | 採用しないか、採用するなら強い理由 |
| 2. 3パターン比較 | 20 | 最低3パターンの比較記録 |
| 3. 商材文脈整合 | 15 | 業種・ペルソナ感性に合う |
| 4. 競合との差別化 | 15 | 競合がダッシュボード型ばかりなら差別化 |
| 5. 採用理由の説得力 | 10 | 5軸採点 + 説明 |
| 6. visual のCV貢献 | 10 | 装飾だけの要素なし |

合計: **100点**

## Passing Score

- **85点以上**: 納品可能
- **75〜84点**: 軽微修正
- **74点以下**: 主要改善

## Critical Fail Conditions

- 1パターンのみ検討
- 「BtoB SaaS だからダッシュボード」の判断
- 「綺麗だから」の主観選定
- ペルソナ無視

## ダッシュボード型 採用条件

採用してよい場合：

```
✅ 採用OK:
- 商材の利用シーンが画面操作で完結
- ペルソナがプロダクト画面で意思決定
- 「画面の見やすさ」が CV理由になる
- 競合と比較して画面UI が圧倒的に優れる
+ 3パターン比較済み
```

採用してはいけない場合：

```
❌ 採用NG:
- 「BtoB SaaS だから」の理由
- とりあえず画面 SS
- 画面UI が業界平均レベル
- 商材が画面操作以外（コンサル / 研修 等）
- 1パターンしか検討してない
```

## Review Questions

- ダッシュボード型を本当に採用すべきか
- 他に商材文脈に合うパターンはないか
- 競合は何を採用しているか
- ペルソナ感性に合うか
