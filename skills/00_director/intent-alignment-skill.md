# Intent Alignment Skill

## Purpose

ユーザーの本来意図と制作中LPの方向性がズレていないか、定期的に確認する。

## When to Use

- 各フェーズ完了時
- 重要意思決定時
- 大きな方向転換が発生したとき

## Required Inputs

- ユーザー依頼文（原文）
- 確定ブリーフ
- 直近フェーズの成果物

## Step-by-Step Procedure

1. ユーザー依頼の「本当に欲しいもの」を再確認
2. 以下9項目をチェック：
   - ユーザーが本当に欲しいLPか
   - CV地点に直結しているか
   - ターゲットがズレていないか
   - 過剰に派手なだけになっていないか
   - デザインが目的より先行していないか
   - 実装できない提案になっていないか
   - 計測設計が抜けていないか
   - 法務リスクを無視していないか
   - 証拠のない実績訴求をしていないか
3. ズレが発見された場合、以下を実施：
   - ズレの内容を明示
   - 該当フェーズに差し戻し
   - 必要ならユーザーに確認

## Output Format

```markdown
## Intent Alignment Check
- 確認日:
- 確認フェーズ:

### 9項目チェック
- [ ] ユーザー意図と一致
- [ ] CVに直結
- [ ] ターゲット整合
- [ ] 派手すぎず
- [ ] 目的先行
- [ ] 実装可能
- [ ] 計測あり
- [ ] 法務OK
- [ ] 証拠あり

### ズレが発見された場合
- 内容:
- 該当フェーズ:
- 対応:
```

## Quality Criteria

- 全9項目を確認
- ズレ発見時に必ず差し戻し
- ユーザー確認が必要な場合は明示

## Common Failure Patterns

- フェーズ完了時のチェックを省略
- ズレを発見しても「軽微」として進める
- ユーザーへの確認を怠る

## Self Check

- [ ] 全9項目を確認したか
- [ ] ズレ発見時の対応を記録したか
