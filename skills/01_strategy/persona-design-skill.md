# Persona Design Skill

## Purpose

ターゲットを「実在する人物」レベルに具体化し、メイン / サブペルソナを定義する。

## When to Use

- 戦略フェーズ
- 訴求軸決定の前段

## Required Inputs

- 確定ブリーフ
- 既存顧客データ（あれば）
- 営業ヒアリング情報

## Step-by-Step Procedure

1. ブリーフから「主ターゲット」を確認
2. メインペルソナを以下の構造で定義：
   - 名前（仮）
   - 業種・職種・役職
   - 年齢・性別
   - 業務内容
   - 業務上の課題
   - 感情上のジョブ
   - 購買決定要因
   - メディア接触
3. サブペルソナを最大2人定義
4. BtoB案件は意思決定者構造（担当 / 上長 / 決裁）を整理
5. 共感マップ（悩み / 欲求 / 不安 / 反論）を作成
6. ペルソナごとの訴求要素を整理
7. `outputs/02_strategy/persona.md` に保存

## Output Format

`templates/lp-strategy-template.md` のペルソナ部分に準拠。

## Quality Criteria

- 「実在する人物」として読める粒度
- 業務上のジョブと感情上のジョブが両方明示
- 訴求軸との対応関係が明確
- BtoBは意思決定者構造あり

## Common Failure Patterns

- 「30代男性」程度の浅さ
- 自社願望ターゲット（実在しない理想顧客）
- 検証なき年収・属性の断定
- 単一ペルソナ（BtoBで複数決裁者を無視）

## Self Check

- [ ] 業務上 + 感情上のジョブを両方記載
- [ ] BtoBなら意思決定者構造あり
- [ ] 訴求軸との対応が明示
