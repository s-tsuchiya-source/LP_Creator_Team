# First View Design Skill

## Purpose

LPの最重要セクション「ファーストビュー」を、3秒で価値が伝わる構成に設計する。

## When to Use

- 構成フェーズ序盤
- 既存LPのFV改善時

## Required Inputs

- 戦略書（訴求軸 / ペルソナ）
- オファー設計
- ヘッドライン候補

## Step-by-Step Procedure

1. 訴求軸とペルソナを再確認
2. 3秒で読める情報量に絞る（メイン1行 + サブ1〜2行）
3. メインCTAの配置を決定
4. 信頼バッジ（実績数値 / ロゴ / 認証）を配置
5. ビジュアル種別を決定（写真 / イラスト / モックアップ）
6. PC / SP の差分を設計
7. `outputs/03_structure/first-view.md` に保存

## Output Format

```markdown
## First View

### PC Layout
- 左: テキスト + CTA
- 右: ビジュアル

### Required Elements
- メインキャッチ:
- サブコピー:
- メインCTA:
- 信頼バッジ:
- ビジュアル:

### SP Layout
- ロゴ
- メインキャッチ
- サブコピー
- メインCTA
- 信頼バッジ
```

## Quality Criteria

- 3秒で価値が取れる
- CTAが目立つ
- 信頼バッジあり
- SP表示で破綻しない

## Common Failure Patterns

- 商品名だけのキャッチ
- 抽象的「世界を変える」
- CTA未配置
- 信頼バッジなし
- SP表示で文字過多

## Self Check

- [ ] 3秒テスト合格
- [ ] CTA可視性
- [ ] SP破綻なし
