# Benefit Translation Skill

## Purpose

機能・特徴を、ユーザー視点のメリットに変換する。

## When to Use

- コピー執筆フェーズ
- 機能セクション・解決策セクション作成時
- LP全体のベネフィット訴求の整合性確認時

## Required Inputs

- 商材の機能リスト
- ペルソナ
- ターゲットの業務状況
- 業種 playbook

## Step-by-Step Procedure

### 7段階の変換式

```
[1. 機能]
   ↓
[2. 何が楽になるか]
   ↓
[3. 何が減るか]
   ↓
[4. 何が増えるか]
   ↓
[5. 誰に評価されるか]
   ↓
[6. どんな未来が得られるか]
   ↓
[7. CVする理由]
```

### 実例

**機能**: API連携機能

```
1. 機能: 既存ツールとAPIで連携
2. 何が楽になる: 二重入力が不要、データ同期が自動
3. 何が減る: 入力工数 月20時間 / 入力ミス頻度
4. 何が増える: データ正確性 / 業務時間の余裕
5. 誰に評価される: 上司「データが信頼できる」/ 経営層「報告の即時性」
6. どんな未来: 経営判断のスピード向上、組織のデータ活用文化
7. CVする理由: 二重入力地獄から解放され、戦略業務に集中できる
```

### 手順

1. 全機能をリストアップ
2. 各機能を7段階で変換
3. 重複・類似ベネフィットを統合
4. ペルソナに最も刺さるベネフィットを上位に配置
5. 数値根拠を付与（出典付き）
6. 業種 playbook の優先ベネフィットと照合
7. 最終的に LP コピーへ反映

## Output Format

```markdown
## ベネフィット一覧

| 機能 | 楽になる | 減る | 増える | 評価される | 未来 | CV理由 |
|---|---|---|---|---|---|---|
| API連携 | 二重入力不要 | 入力工数 月20時間 | データ正確性 | 上司・経営層 | データ活用文化 | 戦略業務に集中 |

## 上位ベネフィット（LP優先掲載）

1. ベネフィット1（数値付き）
2. ベネフィット2
3. ベネフィット3
```

## Quality Criteria

- 7段階全てを変換
- 数値根拠あり
- ペルソナの言葉になっている
- 上司・経営層など周囲の評価も含む
- 法務リスクなし

## Common Failure Patterns

- 機能羅列で終わる（変換が浅い）
- 「効率化」「便利」など抽象表現
- 数値根拠なし
- 周囲の評価視点を欠落
- 業種特性無視

## Self Check

- [ ] 全機能を7段階で変換したか
- [ ] 数値根拠（出典付き）があるか
- [ ] ペルソナの言葉になっているか
- [ ] 上位3ベネフィットを LP 反映準備したか
