# Expectation Building Skill

## Purpose

ユーザーの期待感を、誇張せず根拠ある形で高めるコピー・構成を作る。

## When to Use

- 解決策セクション作成時
- 強み・実績セクション作成時
- CTA直前のコピー作成時

## Required Inputs

- ペルソナ
- 数値実績（出典付き）
- 同業種の事例
- 導入フロー

## Step-by-Step Procedure

### 期待感の4要素を必ず含める

```
1. 数値根拠
2. 事例根拠
3. プロセス根拠
4. 第三者根拠
```

### 手順

1. 期待を高めたいセクションを特定（FV / 解決策 / CTA直前）
2. 各セクションに4要素のうち2つ以上を配置
3. 「導入後の変化」を時系列で描く
4. 「個人差・業種差」を明記
5. 過剰な期待を避けるため「多くの場合」「業界平均」表現を使用
6. CV直前で期待を最大化（「資料を見るだけでも価値がある」と伝える）

## Output Format

### 期待感配置マップ

```markdown
| セクション | 4要素 | 期待コピー |
|---|---|---|
| FV | 数値 + 第三者 | 「導入企業1,200社が選ぶ採用管理ツール / Pマーク取得」 |
| 解決策 | 数値 + プロセス | 「最短2週間で導入。専任担当が支援」 |
| 実績 | 数値 + 事例 | 「平均60%削減（自社調べ・100社）」 |
| CTA直前 | プロセス + 事例 | 「資料には同業種事例5社、ROI試算が含まれます」 |
```

### 「導入後の変化」時系列

```markdown
【1ヶ月後】 操作に慣れ、初期効果を実感
【3ヶ月後】 業務工数 60%削減（出典付き）
【6ヶ月後】 経営層から評価、組織変化

※ 効果は個人差・業種差があります
```

## Quality Criteria

- 期待感の4要素のうち2つ以上を配置
- 全数値に出典
- 「個人差・業種差」明記
- 誇大表現なし
- CV直前のハードル下げと連動

## Common Failure Patterns

- 「絶対に成功」など誇大表現
- 数値出典なし
- 期待だけが先行（不安解消なし）
- 「個人差」明記なし
- 期待をCV直前で過剰に煽る

- [ ] 「個人差・業種差」明記
