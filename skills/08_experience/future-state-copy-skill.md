# Future State Copy Skill

## Purpose

導入後の未来をコピーに落とし、ユーザーが「導入後の自分」を具体的に想像できるようにする。

## When to Use

- 解決策セクション後半の作成時
- 「導入後の流れ」セクション作成時
- 「あなたの業務がこう変わる」セクション作成時

## Required Inputs

- ペルソナ（業務状況・心理状態）
- 数値実績
- 事例
- 業種 playbook

## Step-by-Step Procedure

### 4つの未来をコピー化

```
1. 現状の不満
2. 放置した場合の未来
3. 導入後の未来（時系列）
4. 成果が出た後の未来
```

### 手順

1. ペルソナの現状業務をヒアリング（または仮置き）
2. 4つの未来をテキスト化
3. 一人称（「あなたは〜」）で書く
4. 五感に訴える表現
5. 周囲の反応も描く
6. 数値根拠付き
7. 「個人差」明記

## Output Format

### 採用管理SaaS の例

```markdown
## あなたの業務が、こう変わります

### 朝
ダッシュボードを開くだけで、状況が把握できます。
「あ、ここ対応済みだな」と確認するだけ。

### 日中
重要な意思決定に集中できます。
手作業のデータ集計はゼロ。

### 夕方
経営層への報告は、自動レポートをチェックするだけ。
残業せずに帰れます。

### 上司の反応
「楽になったね」「分かりやすい」と評価される。

### 1ヶ月後
業務に慣れ、初期効果を実感
- 業務工数: 約20%削減

### 3ヶ月後
本格的な効果が見え始める
- 業務工数: 約60%削減（自社調べ・100社平均）
- 抜け漏れ: ほぼゼロ

### 6ヶ月後
組織全体での効果実感
- 経営層への報告自動化
- 部門間連携の改善

※ 効果は個人差・業種差があります
```

### 採用LP の例

```markdown
## ここで働く、あなたの1日

### 9:30
朝会で当日のタスクを確認。
リモートでもオフィスでも自由。

### 10:00〜12:00
集中して開発。
モダンな技術スタック（Next.js / Go / GCP）。

### 12:00
ランチはメンバーと（オフィスの場合）。

### 13:00〜15:00
ペアプロやコードレビュー。
「分からない」を気軽に聞ける文化。

### 15:00
1on1（マネージャーと）。
キャリア相談・技術的な悩みなど。

### 17:00
進捗共有。

### 18:00
退社。プライベート時間の確保。

### 1ヶ月後
業務に慣れ、独自タスクが進む。
メンバーとの関係構築。

### 3ヶ月後
主要機能の改善を担当。
技術スタックに習熟。

### 1年後
リーダーポジションへステップアップ。
採用面接への参加。
給与改定。
```

## Quality Criteria

- 一人称で書かれている
- 五感に訴える表現
- 周囲の反応も含む
- 時系列で描かれている
- 数値根拠（出典付き）
- 「個人差」明記

## Common Failure Patterns

- 抽象的な「効率化」「成長」のみ
- 三人称（「ユーザーは〜」）で書く
- 数値根拠なし
- 周囲の反応欠如
- 「絶対に成功」など誇大表現
