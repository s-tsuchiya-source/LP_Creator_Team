# Self Relevance Design Skill

## Purpose

ユーザーに「これは自分向けだ」と3秒以内に思わせる設計を行う。

## When to Use

- FV 設計時
- 課題セクション設計時
- ターゲット明示が必要なとき

## Required Inputs

- ペルソナ
- 業種 playbook
- ターゲットの言語化されていない悩み

## Step-by-Step Procedure

### 自分ごと化の5要素

```
1. ターゲット明示（業種・役職・状況）
2. 悩みの言語化
3. 業種・役職別の共感
4. 自分ごと化見出し
5. 自分が該当するチェックリスト
```

### 手順

1. ペルソナを精査
2. 「言語化されていない悩み」を10個リストアップ
3. その中から3〜5個を選び、悩みリストにする
4. FV にターゲット明示（「中小企業の人事担当者向け」など）
5. 「こんな悩みありませんか」セクションでチェックリスト化
6. 業種・役職別のセクション分岐（必要なら）

## Output Format

### FV のターゲット明示

```markdown
中小企業の人事担当者へ

採用業務を、最大50%削減する採用管理ツール。
スプレッドシート管理を卒業し、応募者対応を自動化。

[無料で資料請求]
```

### 「こんな悩みありませんか」セクション

```markdown
## こんな悩み、ありませんか？

□ 採用業務に月100時間以上を費やしている
□ 応募者管理がスプレッドシートでバラバラ
□ 良い候補者を見落としてしまうことがある
□ 上長への報告に時間がかかる
□ 採用効果の可視化ができていない

ひとつでも当てはまったら、続きをお読みください。
```

### 業種別共感

```markdown
## こんな業界・規模の方に選ばれています

### 中小製造業 (50〜300名)
人事専任が1〜2名で、採用業務に追われている

### IT・SaaS (50〜500名)
エンジニア採用に特化したい

### 専門サービス業 (30〜200名)
属人化した採用を仕組み化したい
```

### 自分ごと化見出しの型

- 「[ターゲット] のための、[カテゴリ]」
- 「[現状] の[ターゲット] へ」
- 「[業界] で[業務] している方へ」
- 「[役職] が、[課題] を解決する方法」

## Quality Criteria

- FV にターゲット明示
- 悩みリスト 3〜5個
- 業種・役職別の共感（必要なら）
- 「自分のことだ」と感じる粒度

## Common Failure Patterns

- ターゲット明示なし
- 抽象的すぎる悩み（「効率化したい」だけ）
- 業界一般論（特性を捉えていない）
- 「みなさん」など広範囲な訴求

- [ ] 悩みリスト 3〜5個
- [ ] 「自分のことだ」と感じる粒度
