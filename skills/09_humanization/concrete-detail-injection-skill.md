# Concrete Detail Injection Skill

## Purpose

抽象的なコピー・ベネフィットに、具体的なディテールを注入して実在感を出す。

## When to Use

- ベネフィットが抽象的になっているとき
- 数値根拠が薄いとき
- 業務シーンが見えないとき

## Required Inputs

- 抽象的なコピー
- 数値実績（出典付き）
- ペルソナの業務状況
- 業種 playbook

## Step-by-Step Procedure

### 3軸の具体化

```
軸1: 数字 → 「効率化」→「月150時間 → 月60時間（60%削減）」
軸2: 場面・時間 → 「使いやすい」→「移動中の電車内でもスマホで対応」
軸3: 人物・役割 → 「お客様」→「中小企業の人事担当者（30〜45歳）」
```

### 詳細手順

#### Step 1: 抽象語をリストアップ

```
- 効率化
- 業務改善
- 最適化
- 多くの企業
- 短期間
- 高品質
```

#### Step 2: 各抽象語を3軸で具体化

例: 「効率化」

```
軸1（数字）: 月150時間 → 月60時間（60%削減）
軸2（場面）: 月曜の朝、ダッシュボードを開くだけで状況把握
軸3（人物）: 中小企業の人事担当者の場合
```

#### Step 3: 出典付きで記載

すべての数値に出典：

```
「平均60%の業務工数削減（自社調べ・100社平均・2025年実績）」
「導入企業1,200社（自社調べ・2026年4月時点）」
「[第三者調査会社] 調べ・対象者300名・2025年12月」
```

#### Step 4: 業務シーンを描く

時系列で業務を再現：

```
【現状】
9:00 出社、メール確認（10分）
9:10 スプレッドシートを開く（5分）
9:15 各求人媒体から応募者を手動コピペ（30分）
9:45 上司への報告書作成（30分）

【導入後】
9:00 出社、ダッシュボードを開く（5秒）
9:05 状況確認（5分）
9:10 残り時間で戦略的業務に集中
```

#### Step 5: 「個人差・業種差」明記

```
※ 効果には個人差・業種差があります
※ 自社調べ・100社平均（2025年実績）
※ 業界・規模・運用状況により変動します
```

## Output Format

### 具体化ログ

```markdown
# Concrete Detail Injection Log

## 抽象語リスト
| 元の語 | 具体化後（数字 / 場面 / 人物） | 出典 |
|---|---|---|
| 効率化 | 月150時間 → 月60時間 / 月曜の朝の業務 / 人事担当者 | 自社調べ・100社平均 |
| 多くの企業 | 1,200社 | 自社調べ・2026年4月 |
| 短期間 | 最短2週間 | 過去30社の平均値 |

## 業務シーン化
- 現状の時系列描写
- 導入後の時系列描写

## 出典確認
- [ ] 全数値に出典
- [ ] 「個人差」明記
```

## Quality Criteria

- 全抽象語が具体化されている
- 数値に出典付き
- 業務シーンが描けている
- 「個人差・業種差」明記

## Common Failure Patterns

- 抽象語を別の抽象語に置き換え
- 数値だけ追加して場面・人物が抽象のまま
- 出典の追加忘れ
- 「個人差」明記忘れ（景表法リスク）

## Self Check

- [ ] 3軸（数字・場面・人物）すべてで具体化
- [ ] 出典明記
- [ ] 「個人差」明記
- [ ] 業務シーン描写
