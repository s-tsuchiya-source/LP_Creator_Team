# Human Tone Rewrite Skill

## Purpose

きれいすぎる文章を、自然な人間らしい文体に書き換える。

## When to Use

- AIっぽい完璧な文章になっているとき
- 余白がなく押し付けがましく感じるとき
- ペルソナの語彙と乖離しているとき

## Required Inputs

- 編集対象コピー
- ペルソナ（語彙・関心・心理）

## Step-by-Step Procedure

### 5つの書き換え手法

```
1. きれいすぎる文章を崩す
2. 本音を入れる
3. 現場の言葉を入れる
4. 抽象語を具体語に変える
5. 読み手が心の中で言いそうな表現を使う
```

### 詳細手順

#### Step 1: きれいすぎる文章を崩す

文法的に完璧すぎる文を、余白のある文に：

```
Before:
「私たちは、お客様の業務効率向上を実現するために、
最適なソリューションを提供することで、
事業成長を支援してまいります。」

After:
「『採用業務に追われて、本来やるべき面接準備が後回し』

そんな声から生まれたサービスです。

[サービス名] が、月150時間の業務を月60時間に減らします。

※ 自社調べ・100社平均・効果には個人差があります」
```

#### Step 2: 本音を入れる

ペルソナの本音を文章に織り込む：

```
- 「失敗したくない」
- 「上司に怒られたくない」
- 「胸を張って報告したい」
- 「業務疲弊から解放されたい」

→ これらをコピーに自然に入れる
```

例：

```
「『あの応募者、返信した？』
上司から聞かれるたびにヒヤヒヤ。
そんな日々から解放されませんか？」
```

#### Step 3: 現場の言葉を入れる

業界・業務の現場で使われる言葉を活用：

```
✅ BtoB SaaS（採用管理）
- 「応募者管理」
- 「経路別CV率」
- 「内定承諾率」

✅ 採用LP（エンジニア）
- 「ペアプロ」
- 「コードレビュー」
- 「1on1」

✅ EC（化粧品）
- 「肌のキメ」
- 「使用感」
- 「ノリが良い」
```

#### Step 4: 抽象語 → 具体語

```
Before: 業務効率を上げます
After: 月150時間 → 月60時間（60%削減）

Before: 業界最高のサポート
After: 平日9-18時、チャットで即応・FAQ動画あり
```

#### Step 5: 読み手が心の中で言いそうな表現

ペルソナが思いそうな言葉を、見出しや文中に：

```
- 「これ、自分のことだ」
- 「分かってくれている」
- 「今のままじゃダメだよね」
- 「資料くらい見てもいいか」
- 「営業されすぎないかな」
```

例（FAQ）：

```
**Q. 営業されすぎないか心配です**

A. ご心配される方が多くいらっしゃいます。
   [サービス名] では：
   - 営業電話はかけません
   - 個別相談はご希望の方のみ
   - メルマガ登録は任意

   「資料を見るだけ」でも、もちろんOKです。
```

## Output Format

### 書き換えログ

```markdown
# Human Tone Rewrite Log

## Before / After
| 元の文 | 書き換え後 |
|---|---|
| 私たちは業界をリードします | 「採用業務に追われて疲弊」を解決するサービスです |

## 入れた本音
- 「失敗したくない」: [配置箇所]
- 「上司に怒られたくない」: [配置箇所]

## 入れた現場の言葉
- 「経路別CV率」: [配置箇所]
- 「内定承諾率」: [配置箇所]

## 抽象 → 具体
| 抽象 | 具体 |

## 読み手の心の声
- 「これ、自分のことだ」: [どこで響かせたか]
```

## Quality Criteria

- 文章のテンポにメリハリがある
- ペルソナの本音が織り込まれている
- 業界・業務の現場語彙を使用
- 抽象語が具体語に変換済み
- 読み手の心の声を捉えている

## Common Failure Patterns

- 崩しすぎて読みにくくなる
- 本音を入れすぎてネガティブになる
- 業界用語を使いすぎて初心者向けでなくなる
- 「これ、自分のことだ」を直接書きすぎる（押し付け）

## Self Check

- [ ] テンポにメリハリ
- [ ] 本音が自然に織り込まれる
- [ ] 現場語彙が適切
- [ ] 抽象 → 具体 変換済み
