# Natural Flow Editing Skill

## Purpose

セクション間の流れを自然にし、断絶のない物語性を作る。

## When to Use

- セクション間が浮いて見えるとき
- 唐突なセクション転換があるとき
- 読みながら離脱してしまう構成のとき

## Required Inputs

- LP全体構成
- セクション順序
- 各セクションのコピー

## Step-by-Step Procedure

### 4つのチェック

```
1. セクション間の橋渡しがあるか
2. 前のセクションを受けて次に繋がるか
3. 読者の感情ジャーニーが途切れていないか
4. 論理的な流れが断絶していないか
```

### 詳細手順

#### Step 1: 全セクションを通読

LP全体を1度通読し、引っ掛かりを感じる箇所を特定。

#### Step 2: 橋渡しコピーを追加

唐突なセクション転換に橋渡しを：

```
NG:
## 解決策
3つの自動化機能...

## 実績
1,200社が導入...

OK:
## 解決策
3つの自動化機能...

これらが組み合わさることで、すでに多くの企業で効果が出ています。

## 実績
1,200社（自社調べ・2026年4月時点）が...
```

#### Step 3: 感情ジャーニーをチェック

9段階感情ジャーニーで、各セクションの感情を確認：

```
[1. 流入] FV → 興味
[2. 自分ごと化] 課題 → 共感
[3. 課題共感] 課題 → 緊張
[4. 期待] 解決策 → わくわく
[5. 信頼] 実績 → 安心
[6. 不安解消] FAQ → 確信
[7. 行動意欲] 最終CTA → 推進力
[8. CV直前] CTA → 決断
[9. CV後] サンクスページ → 安心
```

各段階の感情が次の段階に繋がっているか確認。

#### Step 4: 論理の流れをチェック

```
✓ 流入意図 → ターゲット明示 → 課題 → 解決約束 → メカニズム → 実績 → 比較 → 価格 → FAQ → CTA

✗ 流入意図 → 機能 → 価格 → 実績 → CTA
（解決約束・課題共感・FAQ が抜けて論理破綻）
```

#### Step 5: 橋渡しコピーの推奨

| 前セクション | 次セクション | 橋渡しコピー例 |
|---|---|---|
| FV | 課題 | 「『業務工数を削減したい』という気持ちは多くが共有しています」 |
| 課題 | 解決策 | 「ひとつでも当てはまったら、続きをお読みください」 |
| 解決策 | 機能 | 「なぜ実現できるのか」 |
| 機能 | 実績 | 「すでに多くの企業で効果が出ています」 |
| 実績 | 比較 | 「具体的に、何が違うのか」 |
| 比較 | 価格 | 「中小企業に最適化したプラン」 |
| 価格 | FAQ | 「ご検討時によくいただくご質問」 |
| FAQ | 最終CTA | 「ご不安が解消されたら、まずは資料を」 |

## Output Format

### フロー編集ログ

```markdown
# Natural Flow Editing Log

## 通読時の引っ掛かり
1. [セクション] と [セクション] の間で唐突感
2. ...

## 追加した橋渡しコピー
| 位置 | 追加コピー |
|---|---|

## 感情ジャーニー検証
| 段階 | 対応セクション | 感情 | 評価 |

## 論理流れ検証
- [ ] 流入意図 → ターゲット明示
- [ ] 課題 → 解決約束
- [ ] メカニズム → 実績
- [ ] 比較 → 価格
- [ ] FAQ → CTA

## 修正履歴
- ...
```

## Quality Criteria

- セクション間の橋渡しがある
- 感情ジャーニー断絶なし
- 論理破綻なし
- 通読してスムーズ

## Common Failure Patterns

- 橋渡しコピーが冗長になる
- 感情ジャーニーの段階を飛ばす
- セクション内で完結させすぎて次に繋がらない

## Self Check

- [ ] 全セクション間に橋渡しがある
- [ ] 9段階感情ジャーニー断絶なし
- [ ] 論理破綻なし
- [ ] 通読でスムーズ
