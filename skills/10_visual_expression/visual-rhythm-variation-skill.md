# Visual Rhythm Variation Skill

## Purpose

LP全体の視覚的リズムを多様化し、読み飽き・離脱を防ぐ。

「全セクション同じ余白・同じカード幅・同じ背景」のテンプレ感を排除する。

## When to Use

- デザインフェーズ
- 既存LPの単調感の改善時
- スクロール離脱が高いとき

## Required Inputs

- セクション構成
- 各セクションの情報量
- ペルソナの読書スタイル

## Step-by-Step Procedure

### Step 1: 視覚的リズムの3要素

```
1. 余白のリズム（密度コントロール）
2. 背景のリズム（色・トーン変化）
3. レイアウトのリズム（左右・中央・グリッド変化）
```

### Step 2: 余白リズム

セクションごとに密度を変える：

```
FV: 余白広め（呼吸する）
課題提起: やや密（緊張感）
解決策: 中（情報整理）
実績: 密（数値の重み）
事例: 余白広め（読みやすさ）
比較表: 密（情報の密度）
価格: 中
FAQ: やや密（読みやすさ）
最終CTA: 余白広め（決断の集中）
```

### Step 3: 背景リズム

```
FV: 白 or 商材ビジュアル
課題: 薄グレー or 警告色トーン落とし
解決策: 白
機能: 白 + アクセント
実績: メインカラーのトーン落とし
事例: 白
比較表: 白
価格: 薄グレー
FAQ: 白
最終CTA: メインカラー + 白文字（強調）
```

3〜4色の背景で交互変化させる。

### Step 4: レイアウトリズム

```
FV: 左右2カラム（PC）
課題: 中央集中（チェックリスト）
解決策: 3カラムグリッド
機能: 左右交互（zig-zag）
実績: 中央 + ロゴ列
事例: 3カラムカード
比較表: 全幅テーブル
価格: 3カラム横並び
FAQ: 中央集中（アコーディオン）
最終CTA: 中央集中
```

連続する2セクションで同じレイアウトを避ける。

### Step 5: NGパターン回避

```
❌ 全セクション 1080px 中央寄せ
❌ 全セクション 角丸カード 3列
❌ 全セクション 同じ余白
❌ 全セクション 白背景
❌ 全セクション 同じフォントサイズ
```

### Step 6: 採用理由を記録

各セクションのリズム選定理由を `outputs/05_design/visual-expression-map.md` に記録。

## Output Format

```markdown
## Visual Rhythm Map

| セクション | 余白 | 背景 | レイアウト | 採用理由 |
|---|---|---|---|---|
| FV | 広 | 白+ビジュアル | 左右2カラム | 呼吸感 |
| 課題 | やや密 | 薄グレー | 中央集中 | 緊張感 |
| 解決策 | 中 | 白 | 3カラム | 情報整理 |
| ... | ... | ... | ... | ... |
```

## Quality Criteria

- 余白の密度が3段階以上
- 背景色が3色以上で交互変化
- レイアウトが3パターン以上
- 連続2セクション同レイアウトなし

## Common Failure Patterns

- 全セクション同じ余白
- 全セクション同じ背景色
- 全セクション同じレイアウト
- 装飾感だけで意味のないリズム変化

## Self Check

- [ ] 余白3段階以上
- [ ] 背景3色以上
- [ ] レイアウト3パターン以上
- [ ] 各リズム選定に理由
