# Headline Rhythm Editing Skill

## Purpose

見出しのリズム（テンポ・余白・強弱）を編集し、読み心地の良いコピーに整える。

## When to Use

- 見出し作成後の編集
- 既存見出しが単調なとき
- 読みにくいヘッドラインの修正

## Required Inputs

- 見出し原案
- ペルソナの読書速度
- セクション文脈

## Step-by-Step Procedure

### Step 1: リズムの3要素

```
1. 文字数のリズム（短文 / 中文 / 長文の組合せ）
2. 句読点のリズム（「、」「。」の配置）
3. 改行のリズム（pc / sp で適切配置）
```

### Step 2: 文字数の使い分け

セクション別の推奨文字数：

| セクション | 推奨 | 例 |
|---|---|---|
| FV メインキャッチ | 15〜25文字 | 「採用工数を、最大60%削減」 |
| FV サブコピー | 30〜50文字 | 「中小企業の人事担当者向け、採用業務を一元管理する採用管理ツール」 |
| 各セクション見出し | 12〜20文字 | 「3つの自動化機能」 |
| カード見出し | 8〜15文字 | 「応募者を一元管理」 |
| CTAボタン | 10〜15文字 | 「3分で資料を受け取る」 |

### Step 3: 句読点のリズム

#### NG: 句読点なしで詰まる

```
❌ 採用業務を最大60%削減し中小企業の人事担当者の負担を軽減します
```

#### OK: 適切な句読点

```
✅ 採用業務を、最大60%削減。
   中小企業の人事担当者向け。
```

### Step 4: 強弱のリズム

```
✅ 強→弱のリズム
「採用工数を、最大60%削減」（強）
「導入1,200社が選ぶ採用管理ツール」（補足・弱）

✅ 短→長→短のリズム
「これから、採用は変わる。」（短・8文字）
「中小企業の人事担当者向け、採用業務を一元管理する」（長・25文字）
「14日間、無料で。」（短・10文字）
```

### Step 5: 体言止めの活用

```
✅ 動詞終わりばかりにしない

NG: 「採用工数を削減します」「業務を効率化します」「報告を自動化します」
OK: 「採用工数の削減」「業務効率化」「報告の自動化」

または混在：
「採用工数、最大60%削減。」（体言止め）
「中小企業の人事担当者向けの採用管理ツールです。」（動詞終わり）
```

### Step 6: NG パターン

```
❌ 全部同じ長さの見出し
❌ 句読点なしで詰まる
❌ 全部「〜します」で終わる
❌ 全部「FEATURE」「ABOUT」等の英語見出し
❌ 改行が不自然（助詞分断）
```

### Step 7: 修正例

#### 修正前
```
私たちは中小企業の人事担当者向けに最適なソリューションを提供することで採用業務を効率化します
```

#### 修正後
```
採用工数を、最大60%削減。

中小企業の人事担当者向け、
採用業務を一元管理する採用管理ツール。

導入1,200社（自社調べ・2026年4月）。
※ 効果には個人差があります
```

### Step 8: 採用理由を記録

```markdown
## 見出しリズム編集

### 修正前
[長い詰まった見出し]

### 修正後
[短→長→短のリズム]
- 短: 「採用工数を、最大60%削減」（強・15文字）
- 長: 「中小企業の人事担当者向け、採用業務を一元管理する採用管理ツール」（補足・35文字）
- 短: 「導入1,200社」（信頼バッジ・8文字）

### 採用理由
- 強弱のリズムで読み心地向上
- 体言止め+動詞終わり混在
- 句読点配置で「読み速度」コントロール
```

## Output Format

`outputs/05_design/japanese-line-break-review.md` に編集記録。

## Quality Criteria

- 文字数の使い分け
- 句読点の適切配置
- 強弱のリズム
- 体言止めと動詞終わりの混在
- 改行の自然さ

## Common Failure Patterns

- 全見出し同じ文字数
- 句読点なしで詰まる
- 全部「〜します」終わり
- 助詞の途中で改行

## Self Check

- [ ] 文字数の使い分けあり
- [ ] 句読点で読みやすい
- [ ] 強弱のリズムあり
- [ ] 体言止め活用
