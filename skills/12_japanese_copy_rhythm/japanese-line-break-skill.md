# Japanese Line Break Skill

## Purpose

日本語見出しを「意味単位」で改行し、不自然なAIっぽい改行を排除する。

## When to Use

- 全コピー作成時
- 既存コピーの改行見直し時

## Required Inputs

- 全コピー（FV / 各見出し / 本文）
- ペルソナの読書速度
- PC / SP 表示要件

## Step-by-Step Procedure

### Step 1: 改行の基本ルール

```
✅ 改行してOK:
- 文の意味区切り（「。」の後）
- 主語と述語の自然な区切り
- 並列要素の区切り
- 修飾語と被修飾語の自然な区切り

❌ 改行禁止:
- 助詞の途中（「を」「が」「に」の前後で分断）
- 修飾語と名詞の途中（「赤い」と「車」を分ける等）
- 複合語の途中（「採用業務」を「採用」と「業務」に分ける等）
- 慣用句の途中
```

### Step 2: NG パターン集

#### NG-1: 助詞の途中

```
❌ 採用業務を、最大60%
   削減します

→ 「を」と「最大」の間で意味が途切れる
```

#### NG-2: 修飾語と名詞の分断

```
❌ 中小企業の
   人事担当者向けの採用管理ツール

→ 「中小企業の」と「人事担当者向け」が分断
```

#### NG-3: 複合語の分断

```
❌ 採用
   管理ツール

→ 複合語「採用管理」が分断
```

#### NG-4: ランダム改行

```
❌ 私たちの
   サービスは、
   貴社の採用業務を
   革新します。

→ 意味区切りでない改行
```

### Step 3: OK パターン集

#### OK-1: 主語 + 述語の自然な区切り

```
✅ 採用業務を、最大60%削減します
   （自社調べ・100社平均）
```

#### OK-2: 並列要素の改行

```
✅ 中小企業の人事担当者へ、
   採用業務を最大60%削減する採用管理ツール
```

#### OK-3: 短メインキャッチ + 補足

```
✅ 月150時間 → 月60時間。
   採用業務、こう変わります。
```

### Step 4: 文字数別の改行ガイド

| 文字数 | 推奨 | 改行場所 |
|---|---|---|
| 〜15文字 | 改行不要 | - |
| 16〜25文字 | 1箇所改行 | 文の中央付近の自然な区切り |
| 26〜40文字 | 2箇所改行 | 主語/述語/並列の区切り |
| 40文字超 | 文を分ける推奨 | - |

### Step 5: 既存コピーの改行検査

各コピーを以下でチェック：

```
1. 助詞の途中で切れていないか
2. 複合語が分断されていないか
3. 意味単位で改行されているか
4. PC と SP で改行位置が適切か
```

### Step 6: 採用理由を記録

`outputs/05_design/japanese-line-break-review.md` に：

```markdown
## 改行修正一覧

| 元コピー | 修正後 | 理由 |
|---|---|---|
| 採用<br>業務を | 採用業務<br>を | 複合語分断防止 |
| 中小企業の<br>人事担当者 | 中小企業の人事担当者<br>向け | 修飾語分断防止 |
```

## Output Format

`contracts/japanese-line-break-output-contract.md` に準拠。

## Quality Criteria

- 助詞の途中で改行なし
- 複合語の分断なし
- 修飾語と名詞の分断なし
- 意味単位で改行
- PC / SP で適切

## Common Failure Patterns

- ランダム改行（「綺麗に見えるから」だけで改行）
- 助詞の途中で改行
- 複合語の分断
- br タグ乱用
