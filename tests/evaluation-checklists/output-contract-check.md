# Output Contract Check

各成果物が `contracts/` 配下の対応 contract に準拠しているかを検証。

## チェック対象

### Brief
- [ ] `contracts/brief-output-contract.md` の必須セクション全て
- [ ] 仮置き / 確定値 区別あり
- [ ] 法務リスク確認済み

### Strategy
- [ ] `contracts/strategy-output-contract.md` の必須セクション
- [ ] ペルソナが「実在感」あり
- [ ] 訴求軸が差別化されている
- [ ] KPIツリーが連結

### Persona
- [ ] `contracts/persona-output-contract.md` の必須項目
- [ ] 業務上 + 感情上のジョブ
- [ ] BtoB なら意思決定者構造

### Structure
- [ ] `contracts/structure-output-contract.md` の必須セクション（10〜13個）
- [ ] CTA最低3箇所

### Wireframe
- [ ] PC / SP 両方の指示
- [ ] CTA可視性

### Copy
- [ ] `contracts/copy-output-contract.md` の必須セクション
- [ ] 全コピー本文（要約禁止）
- [ ] 出典付き数値
- [ ] FAQ最低5件

### Design
- [ ] `contracts/design-output-contract.md` の必須項目
- [ ] コントラスト4.5:1以上
- [ ] SP方針あり

### Code
- [ ] `contracts/code-output-contract.md` の必須要素
- [ ] セマンティックHTML
- [ ] 計測 data 属性

### Measurement
- [ ] `contracts/measurement-output-contract.md` の必須項目
- [ ] 主要ファネル網羅
- [ ] 個人情報の生送信なし

### QA
- [ ] `contracts/qa-output-contract.md` の必須項目
- [ ] BLOCKER FAIL なし

### Final Delivery
- [ ] `contracts/final-delivery-contract.md` の必須20セクション全て
- [ ] Emotional CVR 5要素含む（16〜20）

### Experience Value
- [ ] `contracts/experience-value-output-contract.md` の必須10セクション

### Emotional CVR
- [ ] `contracts/emotional-cvr-output-contract.md` の必須10セクション

## 採点

各 contract の Acceptance Criteria 達成率を記録。
Rejection Criteria 該当時は不合格。
