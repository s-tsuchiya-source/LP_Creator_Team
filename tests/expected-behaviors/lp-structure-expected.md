# Expected: LP構成 16必須項目網羅

## 期待される構成

### 必須16項目

```
1. 流入意図の受け止め（Traffic Intent Match）
2. ファーストビュー（First View）
3. 誰向けかの明示
4. 得られるメリット
5. 課題・悩みの言語化
6. 放置リスク
7. 解決策の提示
8. 解決メカニズム
9. 具体的なベネフィット
10. 証拠・実績・事例
11. 競合・代替手段との違い
12. 導入・利用の流れ
13. 不安解消FAQ
14. CTA直前の行動理由
15. CV後に何が起きるか
16. 最終CTA
```

### 推奨セクション構成（BtoB SaaS 例）

```
1. ファーストビュー
   - ターゲット明示「中小企業の人事担当者へ」
   - メインキャッチ + サブコピー
   - メインCTA + 副CTA
   - 信頼バッジ（実績数値・認証）
   - プロダクト画面

2. 課題提起
   - 「こんな悩み、ありませんか？」
   - 5問チェックリスト
   - 共感文

3. 解決策
   - 解決約束（核心メッセージ）
   - 3つのアプローチ
   - 数値根拠

4. 主要機能 × ベネフィット
   - 5機能を「ベネフィット → 機能」順で

5. 導入企業ロゴ列
   - 6〜12社（許諾済）

6. 導入事例
   - 同業種・同規模 3〜5社
   - 数値効果（出典付き）

7. 競合比較表
   - 客観的評価軸
   - 自社劣位点も含む

8. セキュリティ・SLA

9. 価格プラン（3プラン）

10. 導入フロー
    - 4〜5ステップ

11. FAQ（5〜10件）
    - 6カテゴリ網羅

12. 最終CTA
    - ベネフィット再訴求
    - ハードル段階別CTA
    - 安心要素
    - CV後の流れ予告
```

### 案件タイプ別省略可能性

```
セミナー / ウェビナー: 競合との違い 省略可
EC / 物販: 「導入・利用の流れ」省略可
ローカルサービス: 「競合との違い」省略可
採用LP: 「競合との違い」省略可
```

省略する場合、`outputs/08_review/lp-structure-validation.md` に理由を必ず記録。

## NG パターン

❌ FV がない、または機能していない
❌ 最終CTA がない
❌ 証拠・実績がない
❌ 不安解消FAQ がない
❌ CV後に何が起きるか完全不明

## 採点基準

`scoring/lp-structure-completeness-scorecard.md` で 85+ 必須。

Critical Fail（点数に関係なく差し戻し）：
- FV / 最終CTA / 証拠 / FAQ / CV後 が完全欠落
