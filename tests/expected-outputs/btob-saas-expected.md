# Expected Output: BtoB SaaS LP

## 期待される必須セクション

```
1. ファーストビュー（ベネフィット型キャッチ + メインCTA + 信頼バッジ）
2. 課題訴求（業務工数・コスト）
3. 解決策（3アプローチ）
4. 主要機能 × ベネフィット（5機能）
5. 導入企業ロゴ列
6. 導入事例（同業種・同規模 3〜5社）
7. 競合比較表
8. セキュリティ・SLA
9. 価格プラン（3プラン）
10. 導入フロー
11. FAQ（5〜10件）
12. 最終CTA
```

## 期待される必須コピー要素

- メインキャッチ: 「[業務] を [改善幅] する [カテゴリ]」型
- サブコピー: 補足機能
- CTA文言: 「無料で資料請求」「無料デモを予約」（ハードル段階別）
- 信頼バッジ: 「導入企業X社（自社調べ・YYYY年）/ ISO27001 / Pマーク」

## 期待される必須数値根拠

- 導入企業数（出典付き）
- 業務工数削減率（自社調べ・対象社数）
- 継続率（出典付き）
- 全数値に「※個人差・業種差があります」

## 期待される必須 Emotional CVR 要素

### 自分ごと化
- FV ターゲット明示「中小企業の人事担当者へ」
- 「こんな悩みありませんか」セクション（5問チェックリスト）

### 期待感
- 数値根拠（出典付き）
- 同業種事例（3〜5社）
- プロセス根拠（「最短2週間 / 専任担当」）
- 第三者根拠（認証 / 受賞）
- 導入後時系列（1ヶ月 / 3ヶ月 / 6ヶ月）

### 不安解消
- 6カテゴリFAQ（価格 / 導入 / 効果 / 運用 / 社内 / 失敗）
- 「契約後も自由」設計
- 「営業されすぎない」明示

### CV直前
- ハードル段階別CTA（資料請求 / デモ / トライアル）
- 「今動く理由」（実態に即す）
- CV後の流れ予告

### CV後
- サンクスページ設計
- 自動返信メール案
- 関連資料リンク

## 期待される scorecard

- lp-master: **90点以上**
- emotional-cvr: **85点以上**
- benefit-clarity: **85点以上**
- expectation: **85点以上**
- self-relevance: **85点以上**
- legal-risk: **95点以上 / Critical Fail なし**

## 期待される実装要素

- 計測 data 属性付与（CTA / フォーム）
- セマンティックHTML
- レスポンシブ
- フォーム5〜7項目
- LCP < 2.5s

## NG出力例

- 機能羅列のみ（ベネフィット欠如）
- 数値出典なし
- 「業界No.1」根拠なし
- CTA1箇所のみ
- 「お問い合わせ」だけのCTA
- 不安解消セクションなし
- CV後の流れ不明
