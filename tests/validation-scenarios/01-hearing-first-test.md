# Test 01: ヒアリングから始まるかテスト

## Purpose

`lp-director` が、LP制作依頼を受けた際に、いきなりLPを作り始めず、最大12問以内でヒアリングから始めるかを検証する。

## Scenario

LP制作依頼を最小情報で投入：

```
LP_Creator_Team の lp-director を起点に、
架空の BtoB SaaS「採用管理クラウド RecruitFlow」の資料請求LPを作ってください。
```

詳細情報（ペルソナ・実績・競合・期日 等）は意図的に渡さない。

## Expected Behavior

`lp-director` の初回応答が以下のどちらかになる：

### 期待動作 A: ヒアリングを実施

```
ご依頼ありがとうございます。
最高品質のLPを制作するため、まず12問以内のヒアリングをさせてください。

1. 商材・サービス名は何ですか？（→ RecruitFlow と判明済）
2. LPの目的は資料請求 / デモ予約 / 無料トライアル のいずれですか？
3. 主要ターゲットは誰ですか？（業種・規模・役職）
4. ターゲットが抱えている悩みは何ですか？
5. サービスの主な提供価値は何ですか？
6. 競合・代替手段は何ですか？
7. 強み・差別化要素は何ですか？
8. 実績・事例・数字はありますか？
9. 希望するトーンやデザインイメージは？
10. 法務リスク該当業種ですか？
11. 公開期日はありますか？
12. 出力形式は戦略のみ / コピー込み / 実装込み のどれですか？
```

### 期待動作 B: 仮置き提示 + 確認

ヒアリングなしで仮置き案を提示し、ユーザーに確認する場合も合格：

```
情報が限定的なため、以下を仮置きで進めます。

【仮置き項目】
- ターゲット: 中小企業の人事担当者（30〜45歳）
- CV: 資料請求
- 訴求軸: 採用工数削減
...

進めてよろしいでしょうか？ または、追加情報があればお聞かせください。
```

## Failure Pattern

以下は不合格：

❌ いきなりLPを生成し始める
❌ 12問を超えるヒアリング
❌ 仮置きを明示せず勝手に決め打ち
❌ 重要不明点（CV / 商材 / 法務）を仮置きで進める

## Pass Criteria

- 初回応答にヒアリング or 仮置き提示が含まれる
- 12問以内
- 重要不明点を確認している
- 仮置きの場合は明示している
